亚马逊站内流量体系¶

站内流量的分类¶

自然搜索流量（Organic Search）¶

  • 来自 A9/A10 算法的自然排名
  • 质量高、免费、但不稳定
  • 取决于：相关性 + 转化率 + 账户权重

自然推荐流量¶

  • "看了又看"、"买了又买"等关联推荐位
  • 依赖历史销量和买家行为

广告流量（Paid Search）¶

  • SP（Sponsored Products）
  • SB（Sponsored Brands）
  • SD（Sponsored Display）
  • 见 广告策略

类目/节点流量¶

  • 分类浏览带来的流量
  • 选择精准分类节点是前提

Deals / 促销流量¶

  • LD / BD / DOTD 等活动
  • 短期冲量工具，不宜依赖

站内流量分配的逻辑¶

亚马逊的流量分配遵循一个核心原则：转化率优先

算法判断：同等曝光下，谁的转化率高，就把更多流量给谁。

这意味着：
1. 高转化 Listing 获得滚雪球式的流量增长
2. 低转化 Listing 即使出高价广告也拿不到稳定流量
3. 站内流量的天花板由 选品三维度法 决定——选品阶段已经锁定了流量上限

流量获取的优先级¶

新品期：广告流量为主（冷启动）
成长期：广告 + 自然推荐（提升转化）
成熟期：自然搜索 + 自然推荐为主（降低广告依赖）

与其他概念的关系¶

  • A9A10算法 决定自然搜索流量的分配
  • 广告策略 是站内流量的加速器，可短期购买流量
  • Listing优化 是流量承接的落地页，决定转化率
  • 选品三维度法 决定了类目流量的天花板
