# 程序化 SEO（Programmatic SEO）

> 用一条「抓数据 + AI 生成 + 多语种翻译 + 自动发布」的管线，代替人工写文章，规模化生产 SEO 友好的程序化页面。

## 核心理念

**一句话**：一个人写不过一万人，但一条好管线能写过一万人。

- 传统内容站的瓶颈是**写**（人一天 1-2 篇，一年最多 500 篇）
- 管线站的瓶颈是**管线的稳定性**（爬虫别挂、API 别超额、CDN 别坏）
- 加一种语言 = 翻倍；加一个数据源 = 翻倍 → **线性可规模化**

## 标准管线五步法

| 步骤 | 动作 | 工具 |
|------|------|------|
| 1. 选数据源 | 每天/周自动更新的结构化数据源 | Product Hunt / Github Trending / Hugging Face / Steam / 播客平台 |
| 2. 抓数据 | 自定义爬虫抓结构化字段 | Apify / Bright Data / Python |
| 3. 结构化生成 | 按预设模板填空，**不发挥** | GPT-4o-mini + 结构化 prompt |
| 4. 多语种翻译 | **重写型翻译**（按目标语搜索习惯重组）| GPT-4o / Claude |
| 5. 批量发布 | 静态生成 + 提交 sitemap | Astro / Next.js + Cloudflare Pages |

## 为什么 2026 年还能跑

1. **多语种不对称**：英文 SEO 是红海，但土耳其/越南/印尼语 SEO 内容供给极度稀缺
2. **自动续命的内容池**：每天都有新数据 → Google 持续发现新页面（关键排名信号）
3. **长尾词不受 AI Overview 冲击**："best [X] alternative in [小语种]" 类查询，AI Overview 经常不出现

## 三个最容易踩的坑

### 坑 1：数据源选择错误
- 太热门（被做过几百遍）→ 竞争惨烈
- 太冷门（没搜索量）→ 没人搜
- ✅ 理想数据源特征：**每天更新 + 结构化字段 + 英文红海但小语种稀缺**

### 坑 2：GPT prompt 是"发挥型"而非"结构化"
- ❌ "写一篇关于 X 的文章" → 幻觉 + 东拉西扯 + 被 Google 判低质
- ✅ "按这个结构填空" → 字段包括：产品介绍、用户场景、替代品列表、价格信息

### 坑 3：纯机翻
- ❌ Google Translate 直翻 → 被识别为机翻惩罚
- ✅ GPT/Claude 做**重写型翻译**：按目标语言的**搜索习惯重组**

## 见效周期

- 提交 sitemap 后 **3-6 个月**排名开始出来
- 案例：4 个月做到 Google 月点击 20 万

## 关键案例

- **Product Hunt 管线**：抓取每日产品 → GPT 结构化 → 翻 5-10 种语言 → 静态生成 → 月点击 20 万

## 行动清单

1. 选 3-5 个候选数据源（论坛榜单/行业新品/政府开放数据/行业报告/Github Trending）
2. 选 3-5 个目标小语种（看竞品在那些市场没花精力）
3. 写结构化 prompt（5-7 个字段填空，不发挥）
4. 搭管线：Apify + GPT-4o-mini + Astro/Next.js + Cloudflare Pages
5. 持续监控：sitemap 提交 → 3-6 个月观察收录和排名
