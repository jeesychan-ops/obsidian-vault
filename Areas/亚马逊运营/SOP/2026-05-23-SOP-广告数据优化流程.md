# 亚马逊广告数据优化 SOP — 四步诊断循环

## 核心理念
> 广告优化是持续循环，不是一次性做完的。每周一个小循环，每月一个大循环。

## 流程概览
```
数据采集 → 三率诊断 → 问题定位 → 优化动作 → 效果追踪
```

---

## Step 1: 数据采集（每周固定时间）

**采集频率：**
- 每日：看整体ACOS和花费是否超预算（异常才动）
- 每周一：导出上周广告报告，做完整分析
- 每月末：月度复盘，重大策略调整

**必下载的报告：**

| 报告名称 | 用途 | 下载位置 |
|---------|------|---------|
| 广告活动报告 | 整体ACOS/TACOS/花费/销售额 | SP广告 → 报告 → 广告活动 |
| 搜索词报告 | 关键词转化数据 | SP广告 → 报告 → 搜索词 |
| 投放报告（按ASIN） | ASIN定投转化数据 | SP广告 → 报告 → 投放 |
| 已推广商品报告 | 每个ASIN的SP表现 | SP广告 → 报告 → 已推广商品 |

**数据整理标准：**
每个报告导出后，按以下字段整理：
- 活动名称 / 广告组 / 关键词/ASIN
- 曝光 / 点击 / CTR / 订单数 / CVR / 花费 / 销售额 / ACOS / ROAS

## Step 2: 三率诊断（核心指标体系）

### 2.1 整体健康诊断

| 指标 | 计算公式 | 健康标准 | 预警线 |
|------|---------|---------|--------|
| ACOS | 花费 / 销售额 | 新品<30%，稳定期<20% | >目标ACOS×1.5 |
| TACOS | 广告花费 / 总销售额（含自然） | <15%优秀，<20%合格 | >20% |
| ROAS | 销售额 / 花费 | >3x合格，>5x优秀 | <2x |
| 广告占比 | 广告花费 / 总销售额 | <15%合格 | >25% |

### 2.2 三率分层诊断

**CTR（点击率）诊断：**

| CTR区间 | 判断 | 优化方向 |
|---------|------|---------|
| >1.0% | 优秀，主图/价格有吸引力 | 保持，看转化 |
| 0.5%-1.0% | 正常，主图OK | 优化标题关键词相关性 |
| 0.3%-0.5% | 偏低，主图可能有问题 | 检查主图清晰度/差异化 |
| <0.3% | 危险，主图/价格/排名问题 | 优先优化主图 |

**CVR（转化率）诊断：**

| CVR区间 | 判断 | 优化方向 |
|---------|------|---------|
| >10% | 优秀，Listing转化力强 | 适当加bid抢更多量 |
| 5%-10% | 正常，Listing OK | 持续观察 |
| 2%-5% | 偏低，Listing有问题 | 优先优化Listing（标题/五点/A+） |
| <2% | 危险，流量不精准或Listing差 | 检查关键词相关性+Listing |

**CPC（单次点击成本）诊断：**

| CPC区间 | 判断 | 优化方向 |
|---------|------|---------|
| <$0.5 | 便宜流量，长尾词区域 | 可适当加bid |
| $0.5-$1.5 | 正常区间 | 根据ACOS调整 |
| $1.5-$3.0 | 偏高，核心词竞争激烈 | 找更精准的长尾词替代 |
| >$3.0 | 很高，除非高毛利否则难盈利 | 重新评估关键词策略 |

## Step 3: 问题定位（ACOS诊断三步法）

**第1步：先看整体ACOS，超目标多少？**

- ACOS超出目标≤10%：微调bid（降低5-10%）
- ACOS超出目标10-30%：检查关键词结构
- ACOS超出目标>30%：重新审视Listing和关键词匹配

**第2步：拆解到活动/广告组层级**

- 哪个活动拖累整体ACOS？
- 哪个关键词/ASIN的ACOS最高？
- 高ACOS词占总花费多少？（如果<10%，影响不大）

**第3步：拆解到关键词/ASIN层级**

| 问题类型 | 表现 | 动作 |
|---------|------|------|
| 高曝光低点击 | CTR<0.3%，花费高无转化 | 降低bid 20-30% 或精准否定 |
| 高点击低转化 | CVR<2%，花费高无订单 | 检查关键词相关性 或加入否定 |
| 低曝光高点击 | 展示位置差 | 提高bid 10-20% |
| 零订单词 | 花费>5次点击无订单 | 精准否定 或暂停 |

## Step 4: 优化动作（每周执行清单）

**三问验证（简化）：** ①数据源____ ②结论____ ③依据____

> 验证门来源：[[三问验证门-通用决策核验模板]]

### 4.1 关键词优化（每周）

**调价触发条件（满足任一即可执行）：**
- 单个词花费>$10且ACOS>目标×1.5
- 单个词CTR<0.3%且曝光>1000
- 单个词点击>15次且转化=0

**调价幅度：**
- 降低bid：每次5-20%，不直接暂停（给数据积累机会）
- 提高bid：每次10-20%，不超过建议竞价的1.5倍
- 暂停/否定：花费>30次点击仍无转化

**否定投放：**
- 精准否定：点击>20次无转化的具体词
- 短语否定：某类泛流量词大量低质点击（>100次点击/周，转化<1%）

### 4.2 预算优化（每周）

**判断标准：**
- 每日花费 < 每日预算×70%：预算花不出去 → 检查是否曝光不足
- 每日花费 ≈ 每日预算（频繁撞线）：预算不够 → 提高10-20%

**预算调整原则：**
- 稳定期广告活动不轻易降预算（避免破坏算法）
- 新品期广告活动有数据积累后，每周提高10-20%试探

### 4.3 BID策略调整（每2周评估）

| 当前阶段 | 当前策略 | 切换时机 |
|---------|---------|---------|
| 新品期 | 只降低 | 出现关键词日均点击≥5次且CVR稳定 |
| 成长期 | 固定竞价 | ACOS稳定在目标±10%内 |
| 稳定期 | 动态提高和降低 | 有明确的冲排名目标（如BSR冲Top 50） |

### 4.4 结构优化（每月）

**每月评估项：**
- 自动广告跑出的词，是否有CVR>5%且ACOS<25%的？
  → 立即转入手动精准单独建组
- ASIN定投中是否有ACOS<20%且订单稳定的？
  → 提高bid 10-20%，放大流量
- 是否有活动长期ACOS>目标×2且无法改善？
  → 暂停该活动，重新设计结构

## Step 5: 效果追踪（复盘闭环）

### 周复盘模板

| 维度 | 上周 | 本周 | 趋势 |
|------|------|------|------|
| 整体ACOS |  |  | ↑/↓/→ |
| 整体TACOS |  |  | ↑/↓/→ |
| 总花费 |  |  | ↑/↓/→ |
| 总订单数 |  |  | ↑/↓/→ |
| 平均CPC |  |  | ↑/↓/→ |
| CTR均值 |  |  | ↑/↓/→ |
| CVR均值 |  |  | ↑/↓/→ |

### 月度复盘要点

1. **ACOS达标情况：** 本月ACOS vs 目标ACOS
2. **广告归因：** 广告订单 vs 自然订单比例（理想：广告订单占30-40%，自然订单60-70%）
3. **关键词质量：** 本月新加入的词表现如何？淘汰的词是否正确？
4. **竞品变化：** 竞品是否调整了广告策略？是否需要调整ASIN定投？
5. **下月预算规划：** 根据上月数据和下月目标调整预算分配

## 注意事项

1. **不要只看ACOS：** ACOS低但订单少没有意义，要结合订单量和TACOS综合判断
2. **给数据时间：** 新词/新广告组至少跑2周再做重大调整，算法需要时间学习
3. **避免过度优化：** 每天调整一次是大忌，频繁调整会破坏广告模型
4. **记录每次调整：** 每次调bid/预算/暂停都要记录时间和原因，便于复盘
5. **竞品视角：** 每月看1-2次竞品的广告位和投放词，了解竞争环境变化
