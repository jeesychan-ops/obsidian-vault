## 一、核心观点：为什么SOP变Skill？

### 文章核心观点

> "AI要真正帮你做事，就不能只学你的表达习惯，还要学你怎么判断，先看什么，再排除什么，什么情况下动作可以做，什么情况下必须停下来。"

### SOP vs Skill 区别

| 维度 | SOP（给人看） | Skill（给AI执行） |
|------|--------------|------------------|
| **表达** | 可以模糊，靠经验补 | 必须精确，不能歧义 |
| **判断** | "根据竞品情况优化" | 必须拆解：价格/Review/评分/主图/A+/视频... |
| **边界** | "注意合规" | 必须列出：哪些词不能碰/哪些功效不能写... |
| **执行** | 人可以灵活判断 | AI必须按流程走，不能自由发挥 |

## 二、Skill五层拆解框架

### 第一层：输入标准

**定义**：做这项工作必须有哪些信息，没有就不能出最终建议

```
Listing优化必须输入：
□ 当前阶段（冷启动/爬坡/稳定/衰退）
□ 核心关键词（3-5个）
□ 主要竞品ASIN（3个）
□ 现有Review数量和评分
□ 当前转化率数据
□ 利润率/价格带
□ 合规边界清单
```

### 第二层：判断流程

**定义**：先看什么，再看什么，不能跳步

```
链接问题诊断流程：
1. 流量问题 → 有没有曝光？
2. 点击问题 → CTR低？主图/价格/评分？
3. 转化问题 → CVR低？页面/价格/信任？
4. 信任问题 → Review/QA/品牌？
5. 成本问题 → ACOS高/利润薄？
6. 库存问题 → 断货风险？
7. 竞争问题 → 竞品价格/评分/排名？
```

### 第三层：边界规则

**定义**：哪些情况不能做，哪些动作必须标出来

```
Listing优化边界规则：
❌ 不能承诺疗效/功效
❌ 不能使用最高级词汇（最好/第一/顶级）
❌ 不能包含竞品品牌名
❌ 新品期不能大幅修改标题（权重损失）
❌ 转化率低于类目均值时不能先改文案（先查流量）

⚠️ 数据不足必须标出：
   - Review少于10条时，差异化判断需谨慎
   - ACOS数据不足30天时，不能判断广告好坏
   - 没有对标竞品时，不能确定价格策略
```

### 第四层：输出模板

**定义**：必须按固定结构交付，不能自由发挥

```
【链接诊断报告】

一、问题判断
- 当前阶段：[冷启动/爬坡/稳定/衰退]
- 核心问题：[流量/点击/转化/信任/成本/竞争]

二、证据
- 数据1：[具体数据]
- 数据2：[具体数据]
- 数据3：[具体数据]

三、建议动作（按优先级）
1. [动作1] - 原因：[why]
2. [动作2] - 原因：[why]

四、风险
- 风险1：[描述]
- 风险2：[描述]

五、下一步（需要补充的数据）
□ [缺失数据1]
□ [缺失数据2]
```

### 第五层：复盘迭代

**定义**：每次执行完，把结果写回去，AI持续学习

```
复盘模板：
执行日期：[日期]
执行动作：[动作描述]
预期结果：[描述]
实际结果：[描述]
偏差原因：[分析]
更新判断：[更新到Skill中]
```

## 三、链接诊断工作流程（阿波罗广告框架 + 老魏螺旋打法）

### 阶段判断：先分清在哪里

```
判断维度：
1. BSR排名 → 0-100头部/100-1000爬坡/1000+冷启动
2. Review数量 → <10新品/10-100成长期/>100稳定期
3. 自然流量占比 → <30%依赖广告/30-70%过渡/>70%成熟
4. 毛利率 → <0亏损/0-15%爬坡/>15%盈利期
```

### 阶段对应的运营重心

| 阶段 | 核心目标 | 重点动作 | 评价指标 |
|------|---------|---------|---------|
| **冷启动** | 破零+积累权重 | 低价+广告测款 | 订单量/ACOS |
| **爬坡期** | 冲排名 | 螺旋提价+广告放量 | BSR排名/订单增长 |
| **稳定期** | 保利润 | 降广告+稳价格 | ACOS/毛利率 |
| **衰退期** | 激活/清仓 | 降价/老品激活 | 库存周转 |

## 四、Listing优化工作流程

### Step 1：问题诊断（先于修改）

```
诊断维度（按顺序）：
1. 曝光 → 关键词覆盖够吗？
2. 点击 → 主图够吸引吗？价格有优势吗？
3. 转化 → 页面能说服购买吗？信任够吗？
4. 竞争 → 价格带卡位对吗？

诊断工具：
- 曝光：品牌分析/搜索词报告
- 点击： Listing Attributes / 竞品对比
- 转化： 页面分析 / 竞品差评
- 竞争： 价格监控 / BSR排名
```

### Step 2：竞品分析（必须做）

```
竞品分析维度：
1. 价格分布 → 我应该卡在哪个价格带？
2. Review分析 → 用户夸什么？骂什么？
3. 主图对比 → 我的差异化在哪里？
4. 关键词位置 → 流量词覆盖够吗？
5. 评分分布 → 4.0-4.3有机会超越

分析数量：
- 新品：分析BSR前10名
- 老品：分析直接竞品3-5个 + BSR前10
```

### Step 3：文案优化

```
标题优化：
□ 核心关键词前置（最重要的放最前面）
□ 包含品牌/型号/数量/核心卖点
□ 长度50-200字符
□ 避免：堆砌关键词 / 特殊字符 / 主观词汇

五点优化：
□ 每点一个核心卖点
□ 痛点在前（用户关心什么）
□ 解决方案在后（我怎么解决）
□ 包含规格参数
□ 自然融入SEO关键词

图片优化：
□ 主图：白底、无水印、高清
□ 场景图：展示使用场景
□ 信息图：尺寸/规格/对比
□ 细节图：品质/材质
□ 视频：60秒内核心卖点
```

### Step 4：合规检查

```
合规清单：
❌ 不能写：
   - 最高级词汇（最好/第一/顶级）
   - 未经认证的功效声明
   - 竞品品牌名
   - 绝对化承诺（100%/保证/永远）

✅ 必须有：
   - 产品认证标志
   - 正确的产品参数
   - 符合类目要求的格式
```

## 五、广告分析工作流程

### 阶段判断：广告目标是什么？

```
判断维度：
1. 产品阶段 → 测款期/推广期/放量期/利润期
2. 核心目标 → 测词/冲排名/稳排名/保利润
3. 可接受ACOS → 根据阶段不同容忍度不同

ACOS判断标准（阿波罗框架）：
- 测款期：ACOS < 毛利率即可，重点看数据积累
- 推广期：ACOS = 毛利率 × 1.2-1.5
- 放量期：ACOS < 毛利率 × 1.5
- 利润期：ACOS ≤ 毛利率

❌ 不能只看ACOS高低，要看：
   - 订单量增长趋势
   - 关键词排名变化
   - 自然流量占比
   - TACOS（总销售成本比）
```

### 广告诊断流程

```
诊断维度（按顺序）：
1. 账户结构 → 广告组合/活动/组/关键词结构合理吗？
2. 关键词效果 → 哪些词有转化？哪些烧钱没单？
3. 匹配方式 → 广泛/词组/精确，哪个效率更高？
4. 竞价策略 → 固定/动态/只降低，效果差异？
5. 预算分配 → 钱花在该花的地方了吗？

优化动作判断：
- 点击多没转化 → 检查关键词相关性/落地页
- ACOS太高 → 降竞价/改匹配/否词
- 曝光少 → 提竞价/扩匹配/加词
- 订单稳定但ACOS高 → 优化页面承接
```

### 阿波罗广告架构（整合）

```
四层广告结构：
1. 测词层（自动/广泛） → 积累数据，发现机会词
2. 精准层（手动精确） → 核心转化词，稳定出单
3. 长尾层（手动词组） → 补充流量，降低ACOS
4. 防守层（ASIN定投） → 竞品拦截，防止被抢

预算分配参考：
- 新品：测词层60% + 精准层30% + 长尾层10%
- 成长期：测词层30% + 精准层40% + 长尾层20% + 防守层10%
- 稳定期：精准层50% + 长尾层30% + 防守层20%
```

## 六、老魏螺旋打法 + Skill整合

### 螺旋式爆款启动检查清单

```
启动前必须确认：
□ 盈亏平衡点算清楚了吗？
□ 成本优化到极致了吗？（田忌赛马策略）
□ 备货量够吗？（海运30天 + 安全库存）
□ 广告预算准备好了吗？（$30-50/天）
□ 竞品价格和评分分析了吗？

启动价格判断：
□ 启动价 = 盈亏平衡点（或低$1）
□ 比竞品最低价有竞争力吗？
□ 能支撑多久不亏本？
```

### 螺旋阶段判断

```
判断维度：
1. 销量趋势 → 逐日上升/稳定/下滑？
2. BSR排名 → 持续上升/停滞/下滑？
3. 转化率 → 高于类目平均？
4. 评论积累 → 自然增长？

涨价判断标准：
□ 销量连续7天上升 ✓
□ BSR排名持续提升 ✓
□ 转化率高于竞品平均 ✓
□ 才能考虑涨价

涨价节奏：
- 单次涨幅：$0.5-$1
- 观察周期：3-7天
- 涨价后：必须给足时间观察
```

### 断货预防工作流程

```
日常监控：
□ 库存数量 < 日销量 × 45天 → 触发补货
□ 库存数量 < 日销量 × 30天 → 紧急补货（空运）
□ 库存数量 < 日销量 × 15天 → 断货预警

断货前动作：
1. 小步快跑涨价 → 每天涨$0.2-$0.5
2. 高价自发货跟卖 → FBA价 + $5-$8
3. 降低广告预算 → 省钱等于赚钱

补货后恢复：
1. 适当降价 → 比正常价低$3-4
2. 提高广告竞价和预算 → 配合降价
3. 观察转化率恢复情况
```

## 七、产品开发/VOC分析流程

```
VOC分析流程：
1. 数据收集 → 评论/QA/买家消息/邮件
2. 问题分类 → 痛点类型/频率/强度
3. 优先级判断 → 不是频率高就最重要

痛点优先级判断：
□ 这是强痛点吗？（会导致退货/差评/投诉？）
□ 用户愿意为它付钱吗？
□ 竞品已经解决了吗？
□ 供应链能落地吗？
□ 主图能表达出来吗？

差异化推导：
□ 痛点 → 改款方向 → 供应链评估 → 成本/溢价分析 → 主图表达 → 立项判断
```

## 八、完整工作流程总结

### 日常运营SOP

```
每日必做：
□ 查看广告数据（ACOS/订单/点击）
□ 处理客服消息（24小时内）
□ 检查差评（发现立刻处理）
□ 监控库存（低于45天备货量触发补货）

每周必做：
□ 分析BSR排名变化
□ 关键词排名监控
□ 竞品价格/评分变化
□ 广告结构优化（删/否/调）
□ 库存补货确认

每月必做：
□ 财务核算（毛利率/ACOS/TACOS）
□ 竞品分析更新
□ Listing优化评估
□ 广告预算调整
□ 备货计划复盘
```

### 问题诊断SOP

```
遇到问题时的判断顺序：
1. 这是什么问题？（流量/点击/转化/信任/成本/竞争）
2. 问题出在哪一层？（先判断，不能跳步）
3. 竞品有同样问题吗？（对比分析）
4. 应该先做什么？（不能同时改太多）
5. 改完怎么验证？（设定指标）

❌ 常见错误：
   - CTR低就先改文案（应该先检查主图）
   - ACOS高就降竞价（应该先看转化词）
   - 转化低就改五点（应该先分析差评）
   - 排名下滑就降价（应该先分析原因）
```

## 九、判断逻辑速查表

### Listing优化判断

| 现象 | 原因 | 动作 |
|------|------|------|
| 曝光少 | 关键词不够/排名低 | 查关键词覆盖/开自动广告 |
| CTR低 | 主图不吸引/价格高/评分低 | 优化主图/检查价格 |
| CVR低 | 页面说服力差/信任不够 | 优化五点/看差评 |
| 评分低 | 产品质量/包装/差评 | 查差评/优化产品 |

### 广告优化判断

| 现象 | 原因 | 动作 |
|------|------|------|
| 曝光少 | 竞价低/关键词偏/预算少 | 提竞价/换词/加预算 |
| 点击多无转化 | 关键词不相关/落地差 | 查相关性/优化页面 |
| ACOS高 | 转化低/竞价高/价格低 | 降竞价/优化页面/适当提价 |
| 订单稳定ACOS高 | 流量词偏/页面承接差 | 查搜索词/优化页面 |

### 螺旋打法判断

| 现象 | 判断 | 动作 |
|------|------|------|
| 涨价后跌单 | 正常螺旋/进入价格敏感区 | 继续观察/适当回调 |
| 持续没单 | 启动价太高/产品问题 | 降价/检查选品 |
| 断货预警 | 库存不足 | 立刻补货（空运） |
| 排名下滑 | 断货/差评/竞品上 | 查原因/针对性处理 |

## 十一、团队AI能力建设

### 目标：不只是省人，更要提能力

```
应该问的问题：
□ 产品开发成功率提高了吗？
□ Listing优化更贴近用户需求了吗？
□ 广告动作更符合链接阶段了吗？
□ 团队犯低级错误减少了吗？
□ 新人训练周期缩短了吗？

不应该只问：
❌ 速度提升多少？
❌ 少招了几个人？
❌ 文案外包省了多少钱？
```

## 十二、总结

> "真正拉开差距的，是你有没有自己的流程，有没有自己的判断标准，有没有自己的数据口径，有没有自己的边界，有没有把这些东西训练成AI能执行的工作能力。"

**核心原则**：
1. 先判断问题，再给动作
2. 先分清阶段，再定策略
3. 先有标准，再执行
4. 先能复制，再谈优化
5. 先沉淀经验，再用AI放大
