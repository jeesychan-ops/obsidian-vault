# 品牌自营·笨办法运营工具书

> **适用对象：** 亚马逊品牌自营新手运营
> **核心理念：** "产品不是死在广告里，是死在上架之前"
> **使用方式：** 对着文档一步步做，做完一步再下一步，不要跳步
> **预计总工期：** 上架前准备约45-60天，上架后持续运营

---

## 文档使用说明

### 什么是"笨办法"？

不走捷径、不抄近道。每一步都有标准动作、标准输出、标准判断。做运营最怕的是"差不多得了"，笨办法就是要你每一步都问自己：**这个做到了吗？**

### 文档结构

每一步都统一按以下结构输出：

```
【本步目标】          ← 清楚知道这一步做完要得到什么
【前置条件】          ← 这一步开始前，上一步必须交付什么
【标准动作】          ← 具体操作步骤，一二三列清楚
【工具清单】          ← 这一步用哪些工具，工具怎么用
【输出物】            ← 这一步必须交付的文档/表格
【判断标准】          ← 做完之后怎么判断是否合格
【常见坑】            ← 新手最常犯的错误
【时间参考】          ← 这一步大概需要多少时间
```

## ① 定方向

### 【本步目标】

确定做什么品类、什么市场、用什么策略切入，输出可执行的方向决策书。

---

### 【前置条件】

- 无，上手第一步

---

### 【标准动作】

#### Step 1-1：用工具查市场容量

**工具：Helium 10（推荐）** 或 Jungle Scout（备选）

**具体操作：**

1. 打开 Helium 10 → Black Box（选品数据库）
2. 设置类目过滤器（如 Electronics → Portable Audio）
3. 设置月销量下限为 500/月（筛掉太小的市场）
4. 设置评分范围 4.0-4.5（找到竞争相对弱的机会）
5. 导出前100个结果

**手算天花板（无需工具）：**

```
类目月销量估算：
① 找到BSR#100的产品月销量（如：300单/月）
② 估算整个类目月销量 ≈ BSR#100销量 × 50
③ 年销售额天花板 ≈ 月销量 × 均价 × 12

示例：BSR#100月销300单，类目均价$30
→ 估算月销15,000单 × $30 = 月销售额$450,000
→ 年销售额天花板 ≈ $5,400,000

结论：低于$500万/年的类目，不建议重点做
```

#### Step 1-2：判断竞争度

**打开亚马逊前台，进入目标类目BSR页面**

| 判断维度 | 操作方法 | 判断标准 |
|---------|---------|---------|
| Top10品牌集中度 | 数一数Top10链接，有几个不同品牌 | >50%是同一品牌 → 品牌护城河高，难进 |
| 评论数壁垒 | 记录Top10平均评论数 | 头部>500条 → 高壁垒，要差异化 |
| 新卖家比例 | Top20里有多少是近1年上架的 | <20%新卖家 → 格局稳定，难切入 |
| 价格带分布 | 记下Top20的价格区间 | 有明显价格空白带 → 机会点 |

**记录模板：**

```
类目：__________________
BSR Top20均价：$__________
价格空白带：$__________（暂无或很少产品）
评论壁垒：Top10平均____条（高/中/低）
品牌集中度：Top10有____个不同品牌（品牌/分散）
新卖家机会：____%（有/无明显机会）
```

#### Step 1-3：选择切入策略

根据 Step 1-2 的数据，选择一种策略：

| 策略 | 什么时候选 | 风险 | 适合的产品 |
|------|-----------|------|-----------|
| **差异化** | 有独特功能/设计/专利 | 研发周期6个月+ | 找到竞品明显痛点，且你能解决 |
| **性价比** | 竞品价格偏高，有10%+降价空间 | 利润薄，依赖规模 | 功能和竞品持平，价格低15%+ |
| **细分利基** | 大类目竞争激烈 | 市场天花板低 | 有明确小众人群（如宠物轮椅、残疾人辅具）|

**选策略的自检问题：**

- [ ] 我能在TOP20竞品中找到至少3个明显痛点吗？
- [ ] 我的成本能支撑我定价比竞品低10%还有利润吗？
- [ ] 这个细分市场月搜索量>10,000吗？

三个问题都YES → 可以做。有任何一个NO → 重新评估。

### 【判断标准】

本步合格的标准：

- [ ] 市场年销售额>500万（美国站参考值）
- [ ] Top10评论壁垒<1000条（否则硬冲代价太高）
- [ ] 找到至少一个可切入的价格带或痛点空白
- [ ] 毛利率预估>30%
- [ ] Go/No-Go决策已做出，签字确认

---

### 【常见坑】

| 坑 | 表现 | 解决方法 |
|---|------|---------|
| **凭感觉选品** | "我觉得这个好卖" | 所有判断必须基于数据，没有数据不决策 |
| **只看销量不看竞争** | 选了月销大但评论壁垒极高的品类 | 竞争度分析和容量分析同等重要 |
| **价格算错了** | 毛利算完发现只有15% | 定价=容量×竞品定价参考，成本要反复核 |
| **没定策略就往前冲** | 边做边想，做到一半发现策略不对 | 先选策略，再做词库和竞品 |

- 有经验：1-2天
- 新手第一次：3-5天（需要多轮数据核实）

## ② 关键词词库

### 【本步目标】

建立覆盖80%搜索流量的分层词库——流量词/长尾词/场景词/品牌词/否定词，上一行的词库是后面所有步骤的基础。

---

### 【前置条件】

- ✅ `01-方向决策书` 已完成并签字

---

### 【标准动作】

#### Step 2-1：确定种子词（5-8个）

从方向决策书里提取，或通过以下方式确定：

**找种子词的方法：**

1. 亚马逊前台搜索框下拉词（输入主词，看推荐）
2. 方向决策书里的核心竞品标题（前50个词提取名词）
3. 参考：https://www.google.com/trends （看搜索趋势）

**种子词模板（填入Excel）：**

```
| # | 种子词 | 来源 | 搜索量（月均） |
|---|--------|------|--------------|
| 1 | wireless earbuds | 下拉推荐 | 已知 |
| 2 | bluetooth headphones | 竞品标题 | 已知 |
```

#### Step 2-2：用工具批量扩展关键词

**工具：Sorftime MCP（推荐）** 或 Helium 10 Keyword Generator

**Sorftime MCP操作（我可以直接帮你跑）：**

```
输入：3-5个竞品ASIN（如方向决策书里的主要竞品）
工具：关键词反查
输出：该ASIN的所有流量词+搜索量+竞争度
```

**Sorftime关键词反查 - 操作步骤（自己跑时用）：**

1. 打开 Sorftime → 工具箱 → 关键词反查
2. 输入竞品ASIN（每次最多10个）
3. 选择目标市场（US/UK/DE）
4. 选择时间范围（近30天/近90天）
5. 导出结果为Excel

**Helium 10操作（备选）：**

1. 打开 Helium 10 → Cerebro
2. 输入竞品ASIN
3. 设置过滤器：月搜索量>500，竞争度≠Very High
4. 导出前200个词

**收集关键词的标准：**
- 月搜索量 > 500（太低没流量）
- 竞争度 ≠ Very High（红色太高跳过）
- 去重后保留

#### Step 2-3：关键词分层

打开Excel，按以下规则给每个词打标签：

| 层级 | 定义 | 怎么判断 |
|------|------|---------|
| **流量词** | 月搜>10,000，品类大词 | 所有词里排前20% |
| **长尾词** | 月搜1,000-10,000，需求明确 | 中间50% |
| **场景词** | 含具体场景/人群词 | 含running/bathroom/kitchen/office/pet等 |
| **品牌词** | 含竞品品牌名 | 含Bose/Sony/Apple等 → 加入否定词 |
| **疑问词** | 含what is/how to/why | 可用于QA植入 |
| **否定词** | 低转化/不相关词 | 见Step 2-4 |

**Excel标注方法（示例）：**

```
在Excel新增一列"层级"，对每个词手动判断：
=IF(搜索量>10000,"流量词",
  IF(搜索量>1000,"长尾词",
    IF(OR(ISNUMBER(SEARCH("running",词列)),ISNUMBER(SEARCH("bathroom",词列))),"场景词","长尾词")
  ))
```

#### Step 2-4：建立否定词库

**为什么要做：** 自动广告会跑出很多垃圾词，否定词库帮你提前堵漏

**否定词分类表：**

| 类型 | 否定词示例 | 原因 |
|------|-----------|------|
| 品牌词 | Bose, Sony, Apple, Samsung | 不给竞品送钱 |
| 型号词 | pro, max, plus, mini | 容易跑偏 |
| 用途词 | for kids, child, baby | 人群不符 |
| 材质词 | metal, plastic, silicone | 人群不符 |
| 行为词 | used, refurbished, wholesale, bulk | 不是目标买家 |
| 质量词 | cheap, expensive, free | 低转化 |

**操作：** 在广告后台新建否定关键词组，把以上词用Phrase Negative逐个加入

#### Step 2-5：验证词库覆盖率

**公式：**

```
词库覆盖率 = 目标词月搜索量之和 / 类目总搜索量

操作：
① 取词库里所有词的总月搜索量
② 除以类目总月搜索量（用Helium 10 Magnet估算）
③ 覆盖率>80% → 合格
④ <80% → 补充更多种子词重新扩展
```

### 【常见坑】

| 坑 | 表现 | 解决方法 |
|---|------|---------|
| **只用一个ASIN反查** | 词库覆盖率低 | 至少用3-5个不同定位的竞品ASIN |
| **不建否定词库** | 自动广告花很多钱买垃圾词 | 第一步就建好，广告开了就加进去 |
| **不去重直接用** | 词库里大量重复 | 导出后先按关键词排序去重 |
| **不看搜索趋势** | 选了下降期的词 | 看近6个月Google Trends，平稳或上升才用 |

## ③ 竞品库

### 【本步目标】

抓取目标类目Top20竞品核心数据，建立可查的竞品情报库，后续所有对比和策略都从这里调数据。

---

### 【前置条件】

- ✅ `01-方向决策书` 完成（知道主要竞品）
- ✅ `02-词库` 完成（核心竞品名单从词库反查结果里来）

---

### 【标准动作】

#### Step 3-1：确定竞品名单

**竞品选择标准（符合任一即可入选）：**

- BSR类目前20名
- 同价格带（±30%你的定价）
- 同功能/同使用场景
- 同目标人群

**收集方式：**

1. 打开亚马逊前台，进入目标类目BSR页面
2. 记录前20个ASIN
3. 用关键词在亚马逊搜索，排名前10的也记录
4. 用词库的流量词搜索，记录每页前3个自然位竞品

**ASIN收集模板（Excel）：**

```
| ASIN | 录入日期 | 来源（BSR/搜索/竞品词） |
|------|---------|--------------------|
| B0XXXXX | YYYY-MM-DD | BSR Top10 |
```

#### Step 3-2：数据采集（逐个ASIN）

**每个ASIN采集以下数据：**

| 数据项 | 怎么获取 | 工具 |
|--------|---------|------|
| 月销量 | BSR排名对应估算 | Jungle Scout / Helium 10 |
| 评论数 | 页面右上角 | 直接看 |
| 评分 | 页面左上 | 直接看 |
| 评分趋势 | 近90天评分变化 | Keepa曲线图 |
| 价格 | 页面显示 | 直接看 |
| 上架时间 | 页面详情页或Keepa | Keepa |
| 主图类型 | 人工观察 | 无 |
| 变体数量 | 详情页变体数 | 直接数 |
| FBA/自发货 | 页面Buy Box旁边 | 直接看 |
| 首页评论内容 | 详情页评价Tab | 人工翻阅 |
| 核心关键词 | 标题文本提取 | 人工+工具 |

**Keepa操作（查历史数据）：**
1. 安装Keepa插件或打开 keepa.com
2. 输入ASIN
3. 查看：价格历史/BSR历史/评分历史/上架日期

** Jungle Scout 操作（估销量）：**
1. 输入ASIN
2. 直接读取月销量估算

#### Step 3-3：竞品分析维度

在Excel里新增分析列：

**① 销量稳定性分析**

```
月销量波动 = 最高月销 / 最低月销
- 波动<2倍 → 稳定品
- 波动2-5倍 → 季节性产品
- 波动>5倍 → 脉冲型（靠活动吃饭）
```

**② 差评痛点分析**

```
在每个竞品的评论详情里找1-3星差评
统计高频痛点词：
- 续航不够/充电慢
- 材质廉价
- 连接不稳定
- 尺寸不符
- 图片与实物不符

记录格式：
| 竞品ASIN | 痛点类型 | 出现频率（高频/中/低）| 备注 |
|---------|---------|-------------------|------|
```

**③ 价格节奏分析**

```
找到每个竞品的：
- 日常价：全年大部分时间的价格
- 促销价：LD/7DD/DOTD价格
- 折扣频率：每年参加几次Deal

记录在竞品库价格列
```

**④ 差异化机会点**

```
基于差评分析，整理出：
- 竞品A的痛点 → 我能否解决？
- 竞品B的痛点 → 我能否解决？
- 找到3个以上我能做得更好的点 → 差异化方向
```

## ⑤ Listing文案

### 【本步目标】

写出高转化率、高SEO友好度的完整Listing——标题、五点、描述、Backend Keywords、QA，全部到位，上架即战斗状态。

---

### 【前置条件】

- ✅ `02-词库` 完成（知道哪些词要放进标题/五点）
- ✅ `04-用户画像` 完成（知道痛点和卖点怎么写）

---

### 【标准动作】

#### Step 5-1：标题撰写

**标题黄金公式（亚马逊200字符以内）：**

```
[品牌名] + [核心流量词] + [产品类型] + [核心功能/规格] + [差异化卖点] + [使用场景/人群]

示例（无线耳机）：
TOZONSKY Wireless Earbuds Bluetooth 5.3, 40H Playtime, IPX7 Waterproof,
Noise Cancelling, Touch Control, Sport Running Headphones with Mic

字数统计：每段不超过200字符（含空格）
```

**标题检查清单：**

- [ ] 包含主流量词（前3个字符，含核心词）
- [ ] 字符数<200（含空格和标点）
- [ ] 阿拉伯数字（如40H）优于文字（如Forty Hours）
- [ ] 无特殊字符（只允许 : / - 这三个）
- [ ] 无Emotion词（amazing/great/best等，亚马逊不收录）
- [ ] 品牌名放在最前面（如果已备案）
- [ ] 规格参数完整（容量/尺寸/颜色等）

**不合格标题示例（新手常犯）：**

```
❌ "New 2024 Top Quality Wireless Headphones for Running - Amazing 
Sound - Best Gift for Men Women"（含Emotion词，字符超200）

✅ "TOZONSKY Wireless Earbuds, 40H Playtime, IPX7 Waterproof, 
Bluetooth 5.3, Sport Running Headphones with Mic"
```

#### Step 5-2：五点描述（Bullet Points）

**五点结构（每点格式）：**

```
[功能关键词] + [具体规格/数据] + [用户利益/解决了什么问题] + [使用场景]

示例：
✅ "【40H超长续航】内置高效锂电池，单次使用8小时，充电盒额外
提供32小时续航，一次充电满足全天出行需求，户外运动/通勤无
续航焦虑。"
```

**五点内容规划（参考用户画像）：**

| 五点序号 | 内容方向 | 关键词来源 | 字数控制 |
|---------|---------|----------|---------|
| Bullet 1 | 核心卖点/差异化 | 流量词+场景词 | 200字符以内 |
| Bullet 2 | 痛点解决/功能亮点 | 长尾词+用户痛点 | 200字符以内 |
| Bullet 3 | 规格参数（尺寸/材质/兼容） | 场景词 | 200字符以内 |
| Bullet 4 | 使用体验/易用性 | 好评高频词 | 200字符以内 |
| Bullet 5 | 品牌保障/售后（增加信任）| 通用 | 200字符以内 |

**避开的写法：**

```
❌ 通篇没有关键词（全英文才需要考虑这个，但最好有）
❌ 五点内容重复（同一卖点说两遍）
❌ 规格写错（数字要和实物严格一致）
❌ 堆砌关键词（每个点最多含1-2个主词，不要重复同一个词）
```

#### Step 5-3：产品描述（A+部分）

**产品描述结构（普通卖家可写）：**

```
第一段：品牌故事/产品理念（一句话）
第二段：产品主要特点（3-4句话）
第三段：使用说明/注意事项
第四段：质量保证/售后承诺
```

**有A+权限（品牌备案后）用A+模板，不写纯文字描述。**

#### Step 5-4：Backend Keywords（搜索词）

**规则：**
- 最多249字节（不是字符，是字节，中文1字=2字节，英文1字=1字节）
- 用空格分隔词组，不用逗号
- 不重复标题里已有的词

**填充顺序（按优先级填满为止）：**

```
优先级1：长尾词（还没放进标题和五点的）
优先级2：同义词（product synonyms）
优先级3：应用场景词（补充场景词）
优先级4：相关品类词（扩大曝光）
优先级5：替代品词（客户可能搜的替代方案）
```

**示例（耳机）：**

```
earbuds wireless bluetooth noise cancelling sport running 
exercise gym work office travel android iphone compatible 
gift birthday present
```

#### Step 5-5：QA植入

**找高频问题的方法：**
- 在竞品详情页的QA里找到重复出现的问题
- 在差评里找用户抱怨的"操作复杂/不会用"等问题

**QA写法（用第三方口吻，2-3句话回答）：**

```
Q: Does it work with iPhone 14?
A: Yes, this earbuds uses Bluetooth 5.3, compatible with all 
iOS and Android devices. Works great with iPhone 14 and Samsung 
Galaxy S23.  ← 顺便埋关键词

Q: How long does the battery last?
A: Single charge gives you 8 hours playtime, and the charging 
case provides additional 32 hours. Total 40 hours. Great for 
long flights and outdoor activities.  ← 埋了续航/场景词
```

**QA计划（上架前准备10个Q&A，上架后逐步上传）：**

| # | 问题（顾客视角） | 答案（含关键词） | 上架后状态 |
|---|---------------|--------------|----------|
| 1 | Does it work with...? | ... | 待上传 |
| 2 | How to pair...? | ... | 待上传 |
| ... | | | |

**定价策略选择：**

| 策略 | 做法 | 毛利影响 | 适用场景 |
|------|------|---------|---------|
| **低价抢位** | 比竞品低15%+ | 毛利下降5-10% | 新品冷启动前期，流量匮乏时 |
| **平价进入** | 比竞品低5-10% | 毛利基本持平 | 有差异化卖点，价格非决定因素 |
| **高价定位** | 高于竞品10%+ | 高毛利 | 强差异化/品牌溢价/专利护城河 |

**新品暂定价格 = 竞品均价 × (1-10%)**

#### Step 7-3：毛利测算

```
毛利率 = (售价 - 总成本) / 售价 × 100%

试算表：
| 售价 | 总成本 | 利润/单 | 毛利率 |
|------|--------|--------|--------|
| $19.99 | $12.50 | $7.49 | 37.5% | ← 推荐
| $22.99 | $12.50 | $10.49 | 45.6% | ← 可选
| $17.99 | $12.50 | $5.49 | 30.5% | ← 最低接受线
```

**毛利红线（不可低于）：**

| 客单价 | 最低毛利率 |
|--------|----------|
| <$20 | 25% |
| $20-50 | 30% |
| >$50 | 35% |

**低于以上毛利 → 不建议做，重新评估成本或选品**

#### Step 7-4：定价后期调整计划

```
上架前：暂定价 = 竞品均价×0.9（前3个月抢位用）

上架后3个月评估：
- ACOS<25%，评论>50条 → 提价5-10%
- ACOS>40%，评论<20条 → 维持或降价
- 毛利率持续<25% → 重新评估供应链

年度调价节奏：
- 大促前（LD/DOTD）：降价配合活动
- 大促后：观察是否有必要回调
- 旺季前（Q4）：可根据需求适当提价
```

```
         → 不管转化，先让系统学习

Day 4：  第一次数据检查
         → 点击<20次 → 检查主图/价格/评分
         → 花费>25/天 → 检查竞价是否太高
         → 看到明显的垃圾词 → 立即加否词

Day 5-7：加入手动词组广告（Day 15的步骤提前做）
         → 每天额外$15预算
         → 盯住手动广告转化率

Day 8：  第一次ACOS评估
         → ACOS>40% → 立即降竞价10%+收窄匹配+检查Listing
         → ACOS<30% → 表现良好，可以考虑加预算
```

#### Step 10-3：测单成功/失败判定

**成功标准（达到以下全部）：**

| 指标 | 合格线 | 含义 |
|------|--------|------|
| CTR（点击率）| >0.5% | 主图吸引人 |
| CVR（转化率）| >8% | Listing说服力强 |
| ACOS | <35% | 广告ROI合理 |
| 7天订单 | ≥10单 | 产品有市场 |

**失败处理（任一指标不达标）：**

| 问题 | 症状 | 处理动作 |
|------|------|---------|
| CTR低 | 点击很少或花不出去 | 换主图/调整价格/检查评分 |
| CVR低 | 有点击但没转化 | 检查五点/价格/差评/A+ |
| ACOS爆 | 花费高但转化少 | 降竞价+加否词+检查Listing |
| 断货/不可售 | FBA库存问题 | 立即补货 |

---

### 【判断标准】

| 结果 | 条件 | 动作 |
|------|------|------|
| **大推** | ACOS<25%，CVR>10% | 加预算，冲 |
| **稳住** | ACOS 25-35%，CVR 8-10% | 维持预算，优化Listing |
| **调整** | ACOS>35% 或 CVR<8% | 降竞价+优化Listing，两周后再评估 |
| **止损** | ACOS>50% 持续30天+ | 重新审视选品/定价/供应链 |

## ⑪ 评价积累 + 数据优化

### 【本步目标】

建立评价护城河，持续迭代优化，形成可复制的运营闭环。

---

### 【标准动作】

#### Step 11-1：VINE计划（必做，上架即报）

**什么是VINE？**
亚马逊官方评论获取计划，花钱让亚马逊认证买家留评，评价质量高（真实可信）。

**操作步骤：**

```
亚马逊后台 → 广告 → Vine → 注册产品 → 等待亚马逊分配买家 → 30天后收评价
```

**费用：** $200/ASIN（无论多少条评价，固定费用）

**效果预期：**

| 时间 | 可能收到评价数 |
|------|--------------|
| 30天 | 5-15条 |
| 60天 | 10-25条 |
| 90天 | 15-30条 |

**注意事项：**
- 注册后不可取消
- 评价可能负面（样品质量问题会暴露），先确保产品品质过关再报
- 评分<4.0的VINE评价比没有评价更危险

#### Step 11-2：日常评价管理

**自动请求评价（必须开）：**

```
亚马逊后台 → 设置 → Amazon Vine → 启用"自动请求评价"
```

**差评处理（24小时响应）：**

```
Step 1：在评论下找到"评论者" → 发送站内信（礼貌询问问题）
Step 2：如果买家同意 → 协助退款/换货
Step 3：买家修改/删除差评后，在后台申请"评论争议"（证据充分时可请求移除）
Step 4：差评无法移除 → 用更多好评稀释
```

**差评监控：**
- 每天查看一次评价Tab（新评价提醒要打开）
- 差评出现后立即响应，不要超过24小时

## SOP使用注意事项

### 最重要的三条原则

```
1. 按顺序做，不要跳步
   每一步的输出物是下一步的输入，跳步等于建空中楼阁。

2. 数据说话，不要"我觉得"
   所有判断基于数据，没有数据就去找数据。

3. 宁可少上一个产品，也要保质量
   准备充分的链接，广告打起来才有效。
   仓促上架的链接，边打广告边修，是最贵的做法。
```

### 新手最容易跳过的步骤（必须强制完成）

```
① 词库 - 很多人觉得"差不多词够了"就跳过 → 导致广告打不准
② 竞品库 - 觉得"我看过竞品了"就跳过 → 导致差异化方向错误
③ 用户画像 - 觉得"我知道用户想要什么"就跳过 → 导致文案打歪
④ 否词库 - 上广告了才想起来 → 浪费大量广告费

以上四步，任何一步跳过都必须强制补做。
```
