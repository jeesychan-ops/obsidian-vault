# 亚马逊从0到1标准业务流

> 这是入口文档 — 每个品从选品到稳定出清，按这个流程走。
> 一个品只留1份活的飞书文档，后续更新全部追加，不新建文档。

---

## 总览图

```
P1 选品调研 ──→ P2 市场深挖 ──→ P3 Listing制作 ──→ P4 上架准备
                                                      │
                                                      ↓
                        P7 备货库存 ←── P6 运营优化 ←── P5 广告冷启动
```

| 阶段 | 做什么 | 产出物 | 存放位置 | 用时参考 |
|------|--------|--------|---------|---------|
| **P1 选品调研** | 品类扫描+五维评分，判断进不进 | 选品分析表1行 | 飞书多维表格「选品分析表」 | 1-2天 |
| **P2 市场深挖** | 竞品分析+关键词库+VOC用户洞察 | 竞品报告+词库+VOC笔记 | Obsidian + 飞书 | 3-5天 |
| **P3 Listing制作** | 标题/5点/A+/图片Brief/后端词 | 完整Listing包 | ASIN飞书文档（唯一活的） | 2-3天 |
| **P4 上架准备** | FBA发货/定价/Vine/品牌备案 | 上架检查清单 | 追加到ASIN飞书文档 | 3-7天 |
| **P5 广告冷启动** | 广告架构+预算+关键词投放 | 广告架构+首周预算表 | 追加到ASIN飞书文档 | 持续 |
| **P6 运营优化** | 数据复盘+否定词+ACOS优化 | 周复盘+否定词表 | 追加到ASIN飞书文档 | 每周 |
| **P7 备货库存** | 补货计划+IPI管理 | 补货计划表 | 追加到ASIN飞书文档 | 每月 |

## P1 选品调研

**目标：** 判断一个品类值不值得进

**标准流程：**
1. 确定品类关键词方向
2. 运行选品分析 → 输出五维评分（市场规模/增长/竞争/壁垒/利润）
3. 得分 ≥ 75 → 进入P2；< 75 → 暂存观察池

**使用的技能/工具：**
- `amazon-product-selection` 选品分析技能（自动写入飞书选品分析表）

**产出物位置：** 飞书多维表格 →「选品分析表」
**参考知识库：** [[选品三维度法]]、[[率化选品SOP]]

---

## P2 市场深挖

**目标：** 搞清竞品格局、关键词地图、用户真实需求

**标准流程：**
1. **竞品分析** — Top 10竞品的定价/评论/流量来源/广告打法
2. **关键词库搭建** — 按流量词/转化词/长尾词/品牌词分层
3. **VOC用户洞察** — 7层VOC框架跑一遍（可选，新品可以不做，老品优化必做）

**使用的技能/工具：**
- 竞品分析 → [[竞品分析完整SOP]]
- 关键词库 → [[SOP-关键词库搭建流程]]
- VOC分析 → [[7层VOC框架]]

**产出物位置：** Obsidian（concepts/）+ 关键词表（飞书或Excel）

---

## P3 Listing制作

**目标：** 写出转化率高的Listing

**标准流程：**
1. **标题** — 核心关键词+差异化卖点，留意Rufus推荐机制
2. **五点** — 场景化描述，痛点+解决方案+利益
3. **A+内容** — 品牌故事+情感锚点
4. **图片Brief** — 主图/副图6张/视频脚本
5. **后端Search Terms** — 隐形流量词布局

**使用的技能/工具：**
- `alexa-seo-listing-workflow` — AI辅助Listing重写
- [[7层VOC框架]] — 如果有评价数据，先跑VOC再写

**产出物位置：** **飞书文档 — ASIN命名的唯一活文档**（如 `B0FQJKMLSN Garden Tools V3`）

---

## P4 上架准备

**目标：** 产品上架前把所有坑填好

**检查清单：**
- [ ] FBA货件创建 + 头程发货
- [ ] 定价策略（参考竞品价格带，留出广告空间）
- [ ] Vine计划注册（新品30个）
- [ ] 品牌备案完成
- [ ] 分类节点确认（避免被错分）
- [ ] QA预设（参考VOC Layer 6微吐槽）

**产出物位置：** 追加到ASIN飞书文档

---

## P5 广告冷启动

**目标：** 用可控预算快速积累数据和评价

**标准流程：**
1. **广告架构设计** — 自动广告+手动精准+品牌广告
2. **预算分配** — 按关键词分层（大词试探性、长尾词重点打）
3. **出价策略** — 阶梯出价，动态调整
4. **首周节奏** — 放量测词 → 收数据 → 筛选 → 加预算

**使用的技能/工具：**
- `amazon-apollo-ad-framework` — 阿波罗广告架构
- `amazon-ad-optimization` — 广告优化策略

**产出物位置：** 追加到ASIN飞书文档（广告架构表+否定词表）

---

## P6 运营优化

**目标：** 持续提升ACOS和转化率

**标准流程：**
1. **日** — 盯广告数据，小额调整竞价
2. **周** — 业务报告复盘，否定词整理，搜索词报告分析
3. **月** — ACOS/TACOS复盘，新品方案调整

**否定词管理规范：**
- 统一在飞书「运营分析表」的否定词子表管理
- 每周集中处理一次

**产出物位置：** 追加到ASIN飞书文档
**参考知识库：** [[改Listing后ACOS爆的机制与应对]]、[[广告策略]]

---

## P7 备货库存

**目标：** 不断货、不冗余

**标准流程：**
1. 监控BSR趋势和日均销量
2. 按备货周期（头程+上架时间）计算安全库存
3. IPI < 500 → 清理冗余库存

**参考知识库：** [[财务与利润]]

## 文档规范

### 一个品只留1份活的飞书文档

```
正确做法：
  B0FQJKMLSN Garden Tools（唯一活的）→ 追加V4/V5更新内容

错误做法：
  B0FQJKMLSN V1 方案
  B0FQJKMLSN V2 方案（不要新建）
  B0FQJKMLSN V3 方案（不要新建）
```

### 历史版本处理

旧版Obsidian文件 → 文件名前加 `[历史版本]`，不移除，可追溯。
旧版飞书文档 → 如果老板说"留着当反面教材"就留着，否则合并后归档。
