# Listing 优化体系

## Listing 优化的核心目标

让亚马逊算法（A9/A10）**读懂**你的产品，同时让买家**信任并购买**你的产品。

## 七大优化维度

### 1. 标题（Title）
- 核心关键词前置
- 品牌名 + 产品名 + 核心卖点 + 规格
- 字符控制在 200 以内（移动端友好）
- 避免堆砌关键词

### 2. 主图（Main Image）
- 白底，高清，真实产品为主
- 卖点字幕叠加（视频化呈现）

### 3. 五点描述（A+ / Bullet Points）
- 痛点 + 解决方案
- 规格参数清晰
- SEO 关键词自然融入
- 符合 Rufus 推荐机制的场景化描述

### 4. 搜索词（Backend Keywords）
- 隐藏的额外关键词布局
- 补充标题和描述未覆盖的相关词

### 5. 分类节点
- 选择精准的分类节点获取初始流量

### 6. A+ 内容
- 品牌故事 + 产品对比图 + 使用场景
- 提高停留时间和转化率

### 7. 图文评测（图文并茂）
- 真实使用场景图

## 与算法和其他策略的关系

- Listing 是 [[A9A10算法]] 相关性得分的载体：算法根据 Listing 内容判断展示给谁
- Listing 质量直接影响 [[广告策略]] 的效果：落地页差，广告点击率再高也不转化
- Listing 优化好了，[[站内流量]] 的自然流量部分才会起来
- [[评论与QA]] 是 Listing 社会证明的关键，与 Listing 互相强化
