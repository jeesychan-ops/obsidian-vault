# V10实战工作流：从选品到飞轮的5阶段打法

> **核心理念**：运营不是从上架那天开始的，而是从决定做什么产品那天就开始了。
> 前 500 单之前不要讲宏大规划，先验证 Product-Market Fit。
>
> 整合来源：拓跋勇V10全链路手册（跨境电商长期主义）+ 自有知识库交叉验证

---

## 一、上架前准备（D-60 到 D-0）

### 1.0 前置判断：D-44 门控

拿到一个产品方向后，先问自己：**如果不做这5件事，这个产品后边99%会遇到麻烦。**

| 检查项 | 输出物 | 通过标准 |
|--------|--------|---------|
| 词库闭环 | 结构化5层词库表 | 每层至少20个词，有搜索量/竞价数据 |
| 竞品库闭环 | 八维分析表 + 攻击路径 | 至少锁定3个锚定竞品，路径有证据支撑 |
| 画像闭环 | VOC 表 | 至少2个人群画像，有原话证据 |
| 产品 Brief 闭环 | 完整10项Brief | 差异化声明有竞品证据对照 |
| 产品定义表闭环 | 六锚定定义表 | 缺一不可，三方确认已签字 |

**判断结果**：✅ Go → ⏸️ Pivot → ❌ Kill

---

### 1.1 词库建设（5层分层）

词库不是"给Listing堆关键词"，而是**支撑四大模块的底层数据资产**：Listing撰写、广告架构、视觉方向、否定投放。

**5层词库标准：**

| 层级 | 示例 | 用途 | 搜索量参考 |
|------|------|------|-----------|
| **L1 品牌词** | 自家品牌 + 竞品品牌 | SB广告、品牌防御 | — |
| **L2 核心词** | wireless earbuds | 标题必含、精准广告主力 | 10,000+/月 |
| **L3 属性词** | noise cancelling, waterproof | 五点、A+模块、否定投放 | 1,000-10,000/月 |
| **L4 场景词** | for commuting, gym workout | 附图方向、A+场景、Rufus问答 | **Alexa时代最关键词类** |
| **L5 长尾词** | wireless earbuds for small ears | 广告拓量、低ACOS流量源 | 100-1,000/月 |

**工具链流程**：Sorftime/卖家精灵挖掘种子词 → 竞品反查流量词 → 自动广告跑7-14天搜词报告 → ABA品牌分析 → 按5层分类打标

**铁律**：永远按ASIN建库、每周更新一次、否定词库同步维护、场景词是Alexa时代的新金矿。

**关联知识库**：[[V10-第4-8章-词库竞品画像产品定义#第4章：词库建设（Keyword Library）]]

---

### 1.2 竞品库建设（八维分析+攻击路径）

**锁定标准**：按"价格带±20% + 用户任务重合度"锁定3-5个锚定竞品。

**八维分析表**（每个锚定竞品必填）：

| 维度 | 分析内容 | 信息来源 |
|------|---------|---------|
| 1.流量入口 | 自然搜索词Top5 / 广告词Top5 | Sorftime反查 |
| 2.价格策略 | Coupon / 多变体 / 套装打法 | 前台页面 |
| 3.Review分析 | 好评高频词 / 差评高频痛点 | 评论采集 |
| 4.视觉策略 | 主图风格 / 附图顺序 / 视频 | 前台 |
| 5.A+模块 | 使用模块 / 信息层次 | A+页面 |
| 6.广告强度 | 预估日均花费 | 广告位观察 |
| 7.BSR趋势 | 近3个月排名波动 | BSR追踪 |
| 8.库存健康 | In Stock频率 | Keepa/观察 |

**攻击路径推导**：
- **正面突破** → 竞品弱点是产品问题 → 更好产品+差评黑转白文案
- **侧翼夹击** → 竞品定价偏高 → 同类品质低15-20%
- **降维打击** → 竞品没覆盖某个场景 → 针对该场景重新定义

**铁律**：不包含售价差>50%的产品，不包含BSR Top 3的超级头部，锚定Top 10-50的最易替代。

**关联知识库**：[[V10-第4-8章-词库竞品画像产品定义#第5章：竞品库建设（Competitor Library）]]

### 1.3 用户画像（VOC表+翻译规则）

**四渠道采集**：竞品差评（⭐⭐⭐⭐⭐）> 竞品QA（⭐⭐⭐⭐⭐）> 竞品好评（⭐⭐⭐⭐）> VOC报告（⭐⭐⭐⭐⭐）

**VOC表模板（每人群一条）：**

```yaml
用户画像ID: USER-001
人群标签: 健身爱好者 / 25-40岁 / 男性偏多
使用场景: 健身房训练 / 户外跑步 / 通勤路上
购买动机: 运动时不希望有线缠绕 / 需要稳固定位
核心痛点: 运动时会脱落 / 嘈杂环境音质不行
疑虑点: 防水等级不够 / 续航太短 / 价格偏高
原话证据:
  - "They keep falling off when I run" (差评)
  - "Is this waterproof for sweating?" (QA)
  - "Best for gym, not for everyday use" (好评)
```

**卖点翻译规则**：用户原话 ≠ 卖点，但必须从用户原话里提炼：

| 用户原话 | 翻译成卖点 |
|---------|-----------|
| "They keep falling off" | Secure-fit ear hooks design |
| "Battery dies too fast" | 12-hour playtime |
| "Sweat ruined them" | IPX5 waterproof rating |

**核心原则**：不要自己编词汇，从用户原话里取。

**关联知识库**：[[7层VOC框架]]

### 1.4 产品 Brief（10项模板）

Brief是**从"市场研究"到"产品落地"的转换器**：

```
┌─────────────────────────────────────────┐
│ 1. 商品集结论（目标市场/价格带）          │
│ 2. 目标用户画像（人群/场景/动机/痛点）     │
│ 3. 产品形态定义（功能/材质/尺寸/颜色）     │
│ 4. 差异化声明（跟锚定竞品比好在哪）        │
│ 5. 规格参数表（重量/包装/材质/认证）       │
│ 6. 定价区间（安全线/利润线/广告容忍线）    │
│ 7. 包装要求（FBA合规/品牌VI/说明书语言）   │
│ 8. Listing骨架（核心词/场景词/卖点优先级）  │
│ 9. RFQ采购询价单（数量/交期/质量/验货）    │
│ 10. Go/Pivot/Kill条件                     │
└─────────────────────────────────────────┘
```

**原则**：拒绝模糊词（"好材质"不行，必须写"医用级硅胶"）；每个卖点必须有竞品弱点证据；RFQ标注"做不到就别报价"的底线。

---

### 1.5 产品定义表（六锚定+三方确认）

**六锚定法则**（少一个不能进入下一阶段）：

| 锚定 | 必须回答 | 示例 |
|------|---------|------|
| 功能 | 它做什么？ | 无线充电、降噪、防水 |
| 属性 | 它是什么？ | 入耳式耳机、直立式吸尘器 |
| 材质 | 它用什么做的？ | 硅胶耳塞、铝合金外壳 |
| 形态 | 它长什么样？ | 圆形、方形、可折叠 |
| 颜色 | 它有什么颜色？ | 黑色/白色/军绿色 |
| 参数 | 它的数字是什么？ | 电池500mAh、重量50g |

**三方确认门控**（签字后方可进入定价/文案/视觉阶段）：
- ☐ 供应链确认：所有材质/尺寸/认证可批量生产
- ☐ 设计确认：颜色/形态/包装可执行，与品牌VI一致
- ☐ 运营确认（你）：产品定义能支撑Listing卖点和广告词

---

### 1.6 包装与说明书（D-30 到 D-7）

**包装设计双验收规则**：
- **1600×1600验收**：包装必须包含品牌、品类、核心规格、核心图标
- **148×148验收**：缩略图下品牌/品类/核心图标仍可识别
- 包装正面关键词必须与标题/五点/Search Terms 语义一致

**三件套一致性检查**：Listing页面 / 包装盒 / 说明书 → 产品名、核心卖点、颜色、规格、配件、品牌全部对照一致。

**说明书8模块SOP**：
1. 封面（产品名/型号/品牌Logo/产品图）
2. 开箱清单（图示+数量，标注"不含"配件）
3. 产品图示（多角度+尺寸标注 → 解决"尺寸不合"差评）
4. 快速上手指南（3-5步极简流程）
5. 功能介绍（标注**不支持**的功能 → 降低预期错位）
6. 规格参数表（与A+一致）
7. 故障排除FAQ（从竞品差评提取5-10个问题）
8. 安全警告+保修

**先把差评写进说明书**：锚定竞品差评导出→按问题类型分组→说明书模块对应→FAQ直接引用差评原话。

## 二、上架 Listing 撰写（D-15 到 D-0）

### 2.1 标题

**公式**：`[使用场景/人群] + [核心差异点] + [产品名] + [质量/规格] + [人群标签]`

前80字符决定Alexa收录权重，场景词必须靠前。

### 2.2 五点描述

**结构**：每条回答一个口语化问题。场景/人群 + 核心痛点 + 解决方案 + 精准参数 + 适用边界。

5条五点对应5个典型问题：
- Bullet 1：为什么买这个不是买竞品？
- Bullet 2：这个差异化真的值得吗？  
- Bullet 3：能用多久/会坏吗？
- Bullet 4：配件值不值得？
- Bullet 5：是零风险的吗？

### 2.3 A+ / 视觉

A+是AI可读取的结构化知识库。模块：品牌故事→核心功能→适用人群→使用场景→规格参数→竞品对比→使用步骤。

7张图设计Brief：主图(白底)→场景图→痛点对照→功能细节→尺寸规格→配件清单→使用场景。

**关联知识库**：[[AIO-Listing重写洞察]]、[[Alexa_for_Shopping与答案位]]

## 三、上架后评价 & QA 安排（D+1 到 D+60）

### 3.1 评价积累

**三大渠道**：
1. **Vine计划** → 新品注册30个名额，送测
2. **Insert Card** → 合规话术引导（QR码→Review提交页），禁止要好评
3. **Request a Review** → 订单送达后3-7天触发

**目标**：60天积累20-50条评论，星级维持4.6+。

**场景化评价引导**：在Insert Card提示买家从什么角度写Review（"在什么场景下用了多久？解决了什么问题？"）。

### 3.2 QA 板块

**QA是Alexa问答推荐的核心信息源**。主动设计高频问题并回答：
- 这个产品适合什么尺寸/空间？
- 能不能用于某个具体场景？
- 和普通款有什么区别？
- 有什么限制/不适用的情况？
- 新手怎么用？

**答题规范**：必须用完整陈述句，搭配场景+参数+限制条件，禁止简单Yes/No。

## 四、广告节奏（D+1 到 D+90）

### 4.1 冷启动期（D+1 到 D+30）

**第一阶段：自动广告跑数据**
- 1个自动广告活动（高竞价）
- 跑7-14天获取真实用户搜索词
- 出单词→标记→纳入词库
- 不出单词→加入否定词库

### 4.2 优化期（D+30 到 D+60）

**第二阶段：手动广告拓词+场景词**
- 精准匹配核心词（高出价）
- 短语匹配场景词（中等出价）
- 广泛匹配长尾词（低出价探数据）
- ASIN定投锚定竞品

### 4.3 放量期（D+60 到 D+90）

**第三阶段：飞轮启动**
- 高转化词→扩量（bid+30-50%）
- 自然排名启动→降低广告依赖
- SB品牌广告承接品牌流量
- 场景对话词投放（口语化搜索词）

**关联知识库**：[[阿波罗广告架构]]、[[亚马逊新品从0到1完整作战地图v2.3]]

## 五、飞轮效应（D+30 到 D+90 及以后）

### 5.1 什么是亚马逊飞轮

```
曝光 → 点击 → 转化 → 评价 → 排名 → 更多曝光
                      ↑______________↓
```

### 5.2 前500单之前的核心任务

拓跋勇强调：**前500单之前不要讲宏大规划**。这段时间的精力分配：

| 任务 | 占比 | 说明 |
|------|------|------|
| 流量获取 + 转化率优化 | 40% | 核心目标是验证PMF |
| 评论积累 | 20% | 每单必追Request a Review |
| 广告数据调优 | 20% | ACOS优化+否定词清除 |
| 库存现金流监控 | 20% | 确保不断货、不压货 |

### 5.3 飞轮启动的3个信号

在D+30到D+90这个阶段，持续监控这3个信号：

| 信号 | 含义 | 动作 |
|------|------|------|
| 自然订单占比 >30% | 排名权重已建立 | 逐步降广告预算 |
| ACOS <20%（稳定期） | 广告效率达标 | 扩张精准词 |
| BSR持续上升 | 飞轮已启动 | 加大补货力度+品牌建设 |

### 5.4 止损机制

如果一个产品在D+90时仍没有出现飞轮启动信号，必须回答：
- **词库做对了吗？** → 流量入口覆盖是否足够？
- **竞品分析做对了吗？** → 差异化是否真实？
- **产品定义做对了吗？** → 六锚定是否缺失？
- **定价是否合理？** → 价格带是否支撑冷启动？
- **需要Kill吗？** → 承认产品没达到预期，及时止损

## 五阶段全景图

```
D-60 ─────────────────────────────────────────────────────────── D+90
│                   │                   │                   │
┌─ 上架前准备 ─┐  ┌─ Listing ─┐  ┌─ 评价QA ─┐  ┌─ 广告 ─┐  ┌─ 飞轮 ─┐
│ 词库          │  │ 标题      │  │ Vine计划  │  │ 自动    │  │ 排名   │
│ 竞品库        │  │ 五点      │  │ InsertCard│  │ 手动    │  │ 自然单 │
│ 用户画像      │  │ A+        │  │ RequestRv │  │ ASIN投  │  │ 品牌   │
│ 产品Brief     │  │ 7张图     │  │ QA 设计   │  │ 场景词  │  │ 迭代   │
│ 包装说明书    │  │ 后台属性  │  │ 4.6星+    │  │ 对话词  │  │ 新SKU  │
└───────────────┘  └───────────┘  └───────────┘  └────────┘  └────────┘
```

**关键原则**：
- 前 500 单验证PMF，之后才谈品牌/品类扩张
- 飞轮启动3信号：自然单>30%、ACOS<20%、BSR持续上升
- 止损机制：D+90无信号就复盘，该Kill就Kill
- **运营不是几个动作，而是一条链。链条上任何一个环节太弱，都会把后边的动作拖垮。**
