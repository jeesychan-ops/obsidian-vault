# A9搜索意图六分法

> **核心结论**：Amazon不再是"搜索引擎"，而是**购物行为预测系统**。关键词决定入口，搜索路径决定推荐。

## 一、双模型意图系统

Amazon搜索系统实际由两个模型协同运作：

1. **意图分类模型（Intent Classifier）** — 识别用户搜索背后的真实目标
2. **查询预测模型（Query Predictions）** — 预测用户下一步会搜什么

两个模型相互影响，形成**动态意图网络**：当前搜索结果 → 影响用户下一步搜索词 → 下一步搜索词 → 反过来影响当前搜索排序。

这意味着：**Amazon在主动塑造用户的下一步搜索**，而不是被动响应。

## 二、六种搜索意图类型

| 类型 | 英文 | 含义 | 示例 | 卖家关键动作 |
|------|------|------|------|-------------|
| **细化** | Specification | 用户逐步缩小范围、明确需求 | running shoes → marathon running shoes → Nike marathon running shoes | 覆盖细化词，Listing五点对应每步疑虑 |
| **泛化** | Generalization | 用户放宽条件、扩大搜索 | electric 60 inch standing desk → electric standing desk | 泛化说明结果不满意，需检查转化率 |
| **替代** | Substitution | 用户从当前方向转向竞品/替代方案 | Asics → Nike, retinol cream → CeraVe | A+做非攻击性对比，广告覆盖alternative/similar to |
| **精准查找** | Equivalence | 用户锁定目标品牌/型号 | Asics Gel Nimbus 26 | 精准词卡位，品牌词+型号词布局 |
| **补充** | Complement | 用户寻找配件/耗材/周边 | Brand A Model B → phone case, running socks | "for + 主产品"关键词，配件卡购买后搜索窗口 |
| **无关** | Irrelevant | 当前搜索与此前行为上下文无关 | dog food topper → birthday cards | Listing需明确产品边界，过滤错误流量 |

### 意图之间的跳转逻辑

- Specification转向Substitution：用户连续细化但迟迟不购买 → 系统认为当前方向没满足需求
- Irrelevant出现：用户跳到完全无关类目 → 系统弱化/排除之前的行为上下文
- Complement触发：用户购买或加购某个主产品 → 系统提高配件推荐概率

## 三、搜索不是一个词，而是一条路径

用户搜索不是孤立词，而是一条**连续的行为路径**：

```
running shoes → marathon running shoes → Nike marathon running shoes 
→ Asics marathon shoes → Asics Gel Nimbus 26 → running socks moisture wicking
         ↓                         ↓                    ↓
    Specification             Substitution            Complement
```

路径中用户经历：**大类搜索 → 细化需求 → 品牌偏好 → 竞品替代 → 精准型号 → 配件补充**

卖家任务：
- 理解用户在每一步搜索时的心理状态
- 判断产品在哪一步能切入
- Listing内容是否对应那一步
- 广告是否覆盖关键跳转点

## 四、三个决定流量入口的关键机制

### 1. Complement触发机制
当用户购买/加购主产品后，系统会提高配件推荐概率。配件卖家的黄金窗口不是用户直接搜配件词，而是**用户刚搜完/买完主产品之后**。

### 2. Substitution流失机制
用户连续细化但转化不好 → 系统把用户推向竞品。所以Specification阶段的Listing转化必须足够强：Review支撑、五点回答核心疑虑、A+解释差异化。

### 3. 行为上下文重置机制
用户跳到完全无关类目 → 系统弱化之前的行为上下文。Listing需要在用户进入页面的短时间内完成：**我是谁？我适合谁？我为什么值得买？**

## 五、搜索结果不再统一

同一个关键词，在不同用户、设备、地区或上下文下，触发不同的推荐查询和搜索路径。

**核心结论**：不存在绝对统一的搜索结果页。卖家不能只用单点排名判断流量，要关注：
- SQP中真实用户搜索路径
- 广告中不同词组的转化表现

## 六、Listing优化新逻辑

### 标题：同时覆盖四类词
- Generalization：核心类目词（入口）
- Specification：核心修饰词（细化词）
- Equivalence：品牌/型号/产品名
- Irrelevant过滤：明确产品边界

示例：`Men's Running Shoes - Lightweight Marathon Training Sneakers, Breathable Road Running Shoes`

### 五点：对应用户搜索路径
1. 快速确认产品定位
2. 核心功能/利益
3. 关键规格/材质/参数
4. 使用场景/人群
5. 配件兼容/搭配/售后

### 后台关键词分配建议
| 意图类型 | 建议比例 |
|----------|---------|
| Specification细化词 | 60% |
| Generalization大词 | 15% |
| Equivalence/Substitution精准词与替代词 | 15% |
| Complement配件词 | 10% |

> 不同类目调整：配件类Complement比例更高，品牌类产品Equivalence比例更高，新品牌打竞品流量Substitution要重点研究。

### A+：解决四件事
- 确认：这个产品到底适合谁
- 细化：关键规格和优势
- 替代：为什么选它而不是竞品
- 补充：能和哪些产品一起用

## 七、用户搜索路径地图

把用户从"模糊需求"到"最终购买"再到"购买后补充消费"的完整搜索过程画出来：

```
模糊需求 → 大类关键词 → 细化关键词 → 功能/价格/品牌筛选 
→ 竞品替代 → 精准产品名/型号 → 购买/加购 → 配件/耗材/周边
```

对应六种意图：Generalization → Specification → Substitution → Equivalence → Complement

**搜索路径地图的价值**：不只是告诉"用户会搜什么"，更重要的是：
- 用户什么时候进入高转化阶段
- 用户什么时候开始比较竞品
- 用户什么时候放宽搜索
- 用户什么时候进入配件购买窗口
- 你的Listing应该在哪一步出现

## 十、最终结论

- **关键词优化 = 基础能力**（决定能不能被找到）
- **用户路径优化 = 核心竞争力**（决定会不会被Amazon继续推荐）

未来的赢家，是最懂用户下一步会搜什么的人。
