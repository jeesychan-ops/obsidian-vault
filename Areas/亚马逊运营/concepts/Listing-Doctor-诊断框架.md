# Listing Doctor 诊断框架

> 核心观点：Listing 优化不是内容创作问题，而是**诊断问题**。先搞清楚"为什么不转化"，再决定"改哪里"。

## 一、ASIN 阶段判断（决定优化方向）

| ASIN阶段 | 主要问题 | Listing优化重点 |
|---------|---------|----------------|
| 新品阶段 | 信任不足、点击弱、评论积累少 | 主图点击、价格感、基础卖点、信任信息 |
| 增长阶段 | 流量进入但承接效率不足 | 关键词承接、副图逻辑、五点表达、A+信息顺序 |
| 稳定阶段 | 竞争加剧、同质化 | 差异化表达、品牌感、防守策略、评论优势放大 |
| 衰退阶段 | 页面老化、竞品反超、需求变化 | 竞品复盘、评论问题重拆、卖点重构、价格和促销策略 |

## 二、竞品扫描6维框架

| 拆解维度 | 看什么 | 输出结果 |
|---------|-------|---------|
| 价格 | 价格带、Coupon、Deal、套装价值 | 是否存在价格压力 |
| 评分评论 | 评分、评论数、新增评论、差评主题 | 竞品信任壁垒有多高 |
| 主图 | 是否清楚、突出核心卖点、有点击欲望 | 点击端差距 |
| 副图/A+ | 是否解释价值、场景、尺寸、安装、对比、顾虑 | 转化端差距 |
| 套装/配件 | 是否通过组合形成感知价值 | 差异化切入空间 |
| 品牌感 | 视觉统一度、品牌承诺、售后表达 | 是否适合硬碰硬 |

最终输出：竞品强在哪里 + 弱在哪里 + 可以切入哪里 + 哪些地方不能硬碰硬。

## 三、评论挖掘5类信号

| 评论中反复出现的问题 | 说明什么 | Listing应该怎么改 |
|-------------------|---------|-----------------|
| 安装困难 | 买家担心使用门槛，或实际安装步骤不清楚 | 增加安装步骤图、安装时间说明、配件说明、安装视频 |
| 尺寸不匹配 | 购买前信息不够清楚，适配边界没讲明白 | 增加尺寸确认图、兼容性范围、购买前检查提示 |
| 比想象中更结实（正向） | 可以强化的正向卖点 | 强化材质、承重、耐用性、使用场景表达 |
| 包装一般 | 影响礼品属性和品牌感 | 补充包装展示、礼品场景、开箱体验信息 |
| 使用后有小问题 | 可能带来差评和退货风险 | 在QA、A+或说明图中提前解释使用边界和注意事项 |

> 评论不是售后数据，评论是**用户需求数据**。

## 四、关键词意图分类

| 关键词类型 | 典型表现 | 应该怎么处理 |
|-----------|---------|-------------|
| 高搜索高转化 | 有真实购买需求，也能带来成交 | 进入标题、核心五点和主卖点表达 |
| 高搜索低转化 | 流量大但页面没接住购买意图 | 检查副图、A+、FAQ是否回答用户真正关心的问题 |
| 低搜索高转化 | 长尾但精准，可能更适合当前阶段 | 用副图、A+、FAQ和广告小预算承接 |
| 高竞争词 | 头部垄断明显，当前阶段很难自然排名 | 交给广告测试，不一定强行塞进标题 |
| 无效曝光词 | 有曝光但不点击、不转化 | 降低预算，必要时做否词或弱化内容承接 |

## 五、优化优先级矩阵

| 优先级 | 问题类型 | 处理方式 |
|-------|---------|---------|
| 高优先级 | 直接影响点击、转化、退货或差评 | **马上进入优化任务** |
| 中优先级 | 影响理解、信任、内容完整度 | 进入第二阶段优化 |
| 低优先级 | 视觉锦上添花、表达微调 | 后续迭代处理 |
| 待验证 | 团队内部存在争议，缺少用户证据 | 交给PickFu做真实消费者测试 |

## 六、优化流程总览

```
数据诊断 → 问题分级 → 假设设计 → 用户验证 → 内容生产 → 发布监控 → 持续迭代

角色分工：
- Listing优化skill (诊断) → 找出"为什么不转化"
- PickFu/Market (验证) → 判断哪个方案更有购买吸引力
- Content Studio (生产) → 把策略变成可发布内容
- Operations Assistant (监控) → 判断优化动作是否有效
```
