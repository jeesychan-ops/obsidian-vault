# 三问验证门

> **来源：** 拓跋勇「数据源→结论→判断依据」三段式工作理念
> **用途：** 嵌入所有核心SOP的关键决策节点，防止凭感觉/凭AI给答案就拍板

## 标准验证门

每个关键决策在落地前必须**填写以下三问**，不填完不得进入下一步：

```
┌─────────────────────────────────────────────────┐
│             🔴 三问验证门（强制通过）              │
├─────────────────────────────────────────────────┤
│                                                   │
│  ① 数据源：这个判断基于什么数据？                  │
│     ☐ 亚马逊前台真实搜索数据/搜索结果页             │
│     ☐ 广告报告（≥7天数据积累）                     │
│     ☐ 竞品ASIN反查报告                             │
│     ☐ 真实客户评价/VOC                            │
│     ☐ 个人经验/猜测（❌ 不能作为唯一依据）          │
│                                                   │
│  ② 结论：数据告诉你要做什么决策？                   │
│     （一句话写清楚）                                │
│     → ________________________________           │
│                                                   │
│  ③ 判断依据：为什么这么推导？                      │
│     （写推导逻辑，别人能复现）                      │
│     → ________________________________           │
│                                                   │
│  ✅ 门控检查：                                      │
│     ☐ 数据源≥1个真实来源（非猜测）                  │
│     ☐ 结论与数据源逻辑一致                         │
│     ☐ 推导过程可复现                               │
│                                                   │
│  签名：________  日期：________                    │
└─────────────────────────────────────────────────┘
```

## 简化版（日常小决策用）

```
① 数据源：_______
② 结论：_______
③ 依据：_______
```

> **原则：** 小决策（如竞价微调±10%）可只填简化版，大决策（如选品锁定、广告架构搭建、新品立项）必须填完整版。

## 验证门挂载位置总表

| SOP | 挂载决策节点 | 门类型 |
|-----|------------|--------|

| 新品作战地图 | 各阶段门控检查 | 完整版 |
| 竞品分析SOP | 竞品判定结论输出 | 完整版 |
