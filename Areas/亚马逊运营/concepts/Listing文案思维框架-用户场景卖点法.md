# Listing文案思维框架：用户-场景-卖点法

> 来源：一味君《旧文案写卖点，新文案写用户，转变Listing的思维》（2026-06-01）
> 核心命题：**Listing是销售方案，不是产品说明书**

---

## 核心范式转移

### 旧思维：卖点 → 场景 → 适配用户
- 流量大，泛用户+精准用户都点
- **CVR不稳定 → 流量不稳定 → 出单不稳定 → 排名不稳定 → 广告占比高**
- 适合成熟品抢流量，不适合新品

### 新思维：用户 → 场景 → 卖点
- **只要精准用户，泛流量主动放弃**
- 标题前置筛选 → CVR高 → 可卖高价 → ROI更高
- **新品期必用，泛流量是毒药**

### 一句话判断
> **旧文案是把产品强塞给所有人，新文案是让一类人强烈被吸引。**

---

## 选-开-运三关协同

```
选品（第一关）→ 找精准用户和场景，做验证方案
开发（第二关）→ 把卖点做成解决方案
运营（第三关）→ 文案+视觉+广告触达精准用户
```

**真相：** 运营反复改listing没用的根源是用户和场景在选品阶段就定死了。运营只能在错的根上修枝叶。

## 用户定位三步法

| 步骤 | 动作 | 工具 |
|------|------|------|
| 1. AI生成画像 | 投喂品类给GPT生成用户画像 | ChatGPT/Claude |
| 2. ABA+VOC验证 | 搜索量级 + 真实评论痛点确认 | 卖家精灵/Helium 10 + 评论挖掘 |
| 3. 选定可执行画像 | 排除"想做但够不到"的用户群 | 团队能力评估 |

**AI提示词模板：**
```
我是美国站点的亚马逊卖家，我们的销售产品是[品类]，
请你把购买[品类]的用户画像详细写出来
```

## 场景搭建原则

### 大场景要"压缩"——四同一原则
- 同一地点
- 同一时段
- 同一辆车/同一主体
- 同一光线

❌ **错误：** 城市车主从早到晚一天（时间跨度大，画面像功能拼盘，削弱代入感）
✅ **正确：** 城市停车守护——车停在自己看不见的地方

### 大场景拆小场景
- 每个小场景 = 一个核心痛点
- 每个小场景融入一个卖点
- 围绕一个连贯故事展开

**示例（行车记录仪）：**
- 大场景：城市停车守护
- 小场景1：公寓楼下被前后车夹塞 → 卖点：前后双录
- 小场景2：街边停车过夜被剐蹭 → 卖点：24h停车监控
- 小场景3：停车场出口被撞 → 卖点：G-Sensor碰撞锁定
- 小场景4：夜间车内取证 → 卖点：IR夜视

---

## 卖点开发判断标准

**有效卖点 = 在特定场景中解决用户的必要需求**

### 两个致命误区
1. **功能堆砌式创新**：默认"更多=更好" → 用户愿不愿意多付钱？会不会因为这个选你？
2. **自我感觉式设计**："我觉得用户会这样用" → 中美消费习惯系统性差异

### 避坑方法
- 用户评论挖痛点
- 关键词搜索行为验证
- 竞品分析交叉验证
- **AI结论必须用真实VOC验证，不能直接用**

## 标题布局模板（用户/场景/卖点三前置）

### 案例：三录记录仪

**❌ 旧式（功能堆砌）：**
```
4K+4K+1080P 3 Channel Dash Cam Front and Rear with Cabin, 
Free 64GB Card Included, 5.8GHz WIFI6, IR Night Vision, 
Built-in GPS, G-Sensor, 24H Parking Mode, Support 512GB Max
```

**✅ 新式（用户/场景/卖点前置）：**
```
Taxi Driver 3 Channel Dash Cam                  ← 用户前置
for Passenger and Road Evidence                   ← 场景+购买理由
4K+4K+1080P Front Rear Cabin Camera              ← 解决方案
IR Night Vision, GPS, 5.8GHz WiFi 6, G-Sensor, 
24H Parking Mode                                ← 高价值卖点
64GB Card Included, Max 512GB                    ← 开箱即用
```

**移动端三要素同屏：用户 + 场景 + 卖点 → 第一屏筛掉泛流量**

## 五点描述布局（7条结构）

| 位置 | 内容 | 作用 |
|------|------|------|
| **五点1** | 用户定位 + 主场景 + 核心整体卖点 | 总概括，对号入座 |
| **五点2** | 小场景1 + 卖点1 | 代入感 |
| **五点3** | 小场景2 + 卖点2 | 继续代入 |
| **五点4** | 小场景3 + 卖点3 | 继续代入 |
| **五点5** | 小场景4 + 卖点4 | 继续代入 |
| **五点6** | 竞品对比 + 选择理由 | 解决最后犹豫 |
| **五点7** | 综合补充信息 | 兜底 |

**用户阅读路径：跟着文案代入式思考 → "这就是为我设计的"**

## 价值主张提炼（贯穿选-开-运的主线）

经过用户定位→场景验证→卖点开发，提炼出**一句话价值主张**：

> 案例：针对的士/网约车司机，在夜间场景下，提供一个"无需操作、自动保存、夜间清晰、音频完整、可被平台采信"的行车记录。

**这条主线，所有动作不偏离这句话。**

## AI挖掘场景提示词

**挖掘大场景：**
```
我要在亚马逊美国站销售一款[产品]，目标用户是[人群]，
请你帮我描绘出这个人群与[产品]有关使用场景，
挖掘出用户最无法容忍的问题有哪些，
按照无法容忍的程度进行排序。
```

**挖掘小场景：**
```
围绕场景：[具体场景]。帮我清晰简单的描绘出
用户使用产品过程以及价值感知锚点。
```
