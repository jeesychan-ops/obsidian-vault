# 新品上架前准备：四模块系统性审定框架

## 核心命题

> **很多产品不是死在广告里，而是死在上架之前。**

"先上了再说"这句话，掩盖了上架前应该完成的判断、验证和准备。真正的问题不是"广告没投好"，而是上架前很多事情没有想清楚。

---

## 模块一：产品机会判断

从"找产品"变成"做项目判断"。

### 判断框架三个问题

1. **品类是否真的有机会**
   - 需求是否稳定存在
   - 头部竞品是否已固化
   - 新品是否仍有冒出的可能
   - 市场是否存在**结构性低效**
   - 价格带是否给我们留下利润和广告空间

2. **机会是否能变成我们的机会**
   - 有些机会存在但需要更强供应链
   - 有些需要更高广告预算
   - 有些需要更强内容能力
   - 周期太长/试错成本太高，不适合当前团队

3. **产品是否值得进入下一步验证**
   - 不急着拍板
   - 把"我觉得能做"变成"有哪些证据支持它值得继续看"

### 结构性低效识别

市场不是没有竞争，而是竞争对手在关键环节做得不够好：
- 用户痛点长期没有被解决
- 图片表达和真实使用场景错位
- 评论里反复出现同一类抱怨，但无人处理
- 价格带中间有断层
- 产品规格/包装/功能组合和目标人群之间存在错配

## 模块二：市场调研怎么做

市场调研不是收集信息，而是**形成判断**。

### 必须回答三个问题

| 问题类别 | 具体问题 |
|---------|---------|
| 真实需求？ | 用户是否持续搜索？需求是否季节性/短期热度？不同关键词背后是否同一类需求？需求是否被充分满足？ |
| 竞争结构？ | 头部是否垄断？中腰是否有机会？新品是否有增长样本？Review门槛是否过高？广告位是否极度拥挤？ |
| 有没有空间？ | 价格带是否合理？成本能否承受？Coupon+广告+退货+仓储后还有利润？差异化用户是否愿意买单？ |

**如果答不上来，"我觉得能做"本质上还是感觉。**

---

## 模块三：用户画像怎么落地

用户画像不是漂亮描述，而是**购买决策证据**。

### 无效 vs 有效的用户画像

| 无效描述 | 有效画像要回答 |
|---------|--------------|
| "30岁女性，注重生活品质" | 用户为什么搜这个词？ |
| | 他在买之前担心什么？ |
| | 他在评论里反复夸什么/骂什么？ |
| | 他到底要的是功能、结果、场景、身份，还是情绪上的确定感？ |
| | 他为什么买A不买B？ |
| | 他看到什么图会停下来？ |
| | 他看到什么表达会觉得"这个就是我要的"？ |

### 实际操作方法

把 **VOC（用户评论/Q&A/差评/正评/真实反馈）** 和 **高相关搜索词** 放在一起看：

- 单看评论 → 变成痛点摘要
- 单看关键词 → 变成流量列表
- **两者结合** → 知道用户为什么犹豫、为什么放弃、应该用什么方式承接他

### 输出物

- VOC证据矩阵
- 搜索词意图矩阵
- Persona归因矩阵
- 购买理由与风险回应清单
- 图片/文案/包装/FAQ的表达方向

## 模块四：产品能不能做，怎么审定

> **产品能不能做，不应该只靠老板拍板。**

### 必须摆上桌面的问题

**供应链类：**
- [ ] 供应商是否真的能稳定交付
- [ ] 样品和大货是否一致
- [ ] MOQ能不能承受
- [ ] 打样周期和大货周期能不能匹配
- [ ] 包装、认证、质检有没有风险

**财务类：**
- [ ] 采购成本是否可控
- [ ] 头程/仓储/配送/退货是否算进去
- [ ] Coupon/广告/折扣之后还有没有利润
- [ ] 目标售价有没有足够空间

**决策假设类：**
- [ ] 哪些假设已被验证，哪些还只是猜测
- [ ] 如果验证不成立，什么时候停
- [ ] 如果前期要亏，最多能亏到什么程度

**执行责任类：**
- [ ] 谁负责项目推进
- [ ] 哪些节点必须复盘

### 核心工具

- RFQ矩阵
- 单位经济模型
- 审定会纪要

> **一个团队真正成熟的地方，不是永远说Go，而是有能力在该No-Go的时候No-Go。**

## Day -60 ~ Day 0 作战地图

```
Day -60 ~ -50  机会扫描
               榜单+搜索结果页观察
               初步价格带判断
               初步竞品池建立

Day -50 ~ -40  锚定产品与竞品裂缝
               锚定ASIN库
               Review门槛判断

Day -40 ~ -30  供应链验证与成本测算

Day -30 ~ -25  团队审定会
               供应链证据汇总
               Go / Continue / No-Go

Day -25 ~ -10  预发布准备
               Listing+视觉+定价+广告框架预判
               用户画像落地与购买理由确认

Day -10 ~  0   上线前检查
               Listing + 库存 + 价格 + Coupon + 预算
```

## 上架前必回答的问题清单

在产品真正上架前，必须想清楚：

- [ ] 这个品类为什么值得进入
- [ ] 竞争裂缝到底在哪里
- [ ] 锚定ASIN为什么是它们
- [ ] 用户评论里反复出现的痛点是什么
- [ ] 搜索词背后的购买意图是什么
- [ ] 目标用户为什么会买你而不是买别人
- [ ] 供应商/样品/成本/周期是否撑得住
- [ ] 单位经济模型是否经得起广告/Coupon/退货压力
- [ ] 团队是否已形成Go/Continue/No-Go判断
- [ ] 文案/视觉/包装/广告/种草节奏是否已经对齐

> ⚠️ **时间到了，不等于准备好了。**

## 核心金句

1. "少做一个错误项目，有时候比多做一个看起来不错的项目更重要。"
2. "把感觉包装成判断，是做产品最怕的事。"
3. "上架之后的问题，会用真金白银每天提醒你：你当初没有想清楚。"
4. "真正值得做的产品机会，往往不是被发现的，而是被调研、验证和审定出来的。"
