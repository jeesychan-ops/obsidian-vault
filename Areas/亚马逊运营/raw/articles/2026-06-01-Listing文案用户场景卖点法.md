## 核心范式转移（最重要的认知升级）

### 旧文案思维：卖点 → 场景 → 适配用户
- 优点：流量大，泛用户+精准用户都点进来
- 缺点：**CVR不稳定 → 流量不稳定 → 出单不稳定 → 排名不稳定 → 广告占比高**
- 本质：把产品强塞给所有人，转化率必然低

### 新文案思维：用户 → 场景 → 卖点
- 优点：**只要精准用户，泛流量主动放弃 → 标题前置筛选 → CVR高 → 可卖高价**
- 缺点：精准用户需求量小
- 本质：把产品做给一类人，这类人会被强烈吸引

**核心判断：** 新品期一定用新思维，泛流量是毒药，精准用户是解药。

---

## 选品-开发-运营三关协同（君哥定调）

```
选品：找精准用户和用户使用场景（做验证方案）
开发：开发解决用户场景的卖点（做解决方案）
运营：把整套可视化可收录文案通过广告触达给精准用户（做销售方案）
```

**新流程分工：**
- 选品（第一关）→ 提供"用户×场景×需求"三角验证
- 开发（第二关）→ 把卖点做成解决方案
- 运营（第三关）→ 文案+视觉+广告触达

**运营反复改listing没用的真相：** 用户和场景在选品阶段就定死了，运营改文案只是"在错的根上修枝叶"。

## 用户定位方法（运营最简单的实操）

### Step 1：AI生成用户画像
```
提示词模板：
我是美国站点的亚马逊卖家，我们的销售产品是[品类]，
请你把购买[品类]的用户画像详细写出来
```

### Step 2：ABA + VOC 验证
- ABA看搜索量级（真的有这类用户在搜）
- VOC（用户之声）看评论确认（真的存在这类痛点）

### Step 3：选定符合团队能力的画像
- 排除"想做但够不到"的用户群
- 选定**你供应链+流量+资金能覆盖**的精准用户

## 场景搭建的核心原则

### 大场景要"压缩"
- ❌ 错误：城市车主一天从早到晚（时间跨度大，画面像功能拼盘，削弱代入感）
- ✅ 正确：**同一地点、同一时段、同一辆车、同一光线**

### 大场景拆小场景
- 围绕一个连贯故事展开
- 每个小场景 = 一个核心痛点
- 每个小场景融入一个卖点

**示例（行车记录仪）：**
- 大场景：城市停车守护（车停在自己看不见的地方）
- 小场景1：公寓楼下被前后车夹塞 → 卖点：前后双录
- 小场景2：街边停车过夜被剐蹭 → 卖点：24h停车监控
- 小场景3：停车场出口被撞 → 卖点：G-Sensor碰撞锁定
- 小场景4：夜间车内取证 → 卖点：IR夜视

## 卖点开发原则

**判断标准：**
- ❌ 卖点的有效性不取决于技术难度
- ✅ 卖点有效性 = 是否在特定场景中解决用户的**必要需求**

**两个致命误区：**
1. **功能堆砌式创新**：默认"更多=更好"，用户愿不愿意为这个功能多付钱？会不会因为这个功能选你？
2. **自我感觉式设计**："我觉得用户会这样用" → 中美消费习惯系统性差异，从评论就能直观感受

**避坑方法：** 用户评论里挖 → 关键词搜索行为验证 → 竞品分析交叉验证

## 标题布局模板（用户/场景/卖点三前置）

### 案例对比（三录记录仪）

**❌ 旧式标题（功能堆砌）：**
```
4K+4K+1080P 3 Channel Dash Cam Front and Rear with Cabin, 
Free 64GB Card Included, 5.8GHz WIFI6, IR Night Vision, 
Built-in GPS, G-Sensor, 24H Parking Mode, Support 512GB Max
```

**✅ 新式标题（用户/场景/卖点前置）：**
```
Taxi Driver 3 Channel Dash Cam                  ← 用户前置
for Passenger and Road Evidence                   ← 场景+购买理由
4K+4K+1080P Front Rear Cabin Camera              ← 解决方案
IR Night Vision, GPS, 5.8GHz WiFi 6, G-Sensor, 
24H Parking Mode                                ← 高价值卖点补充
64GB Card Included, Max 512GB                    ← 开箱即用
```

**手机端三要素同时展示：用户 + 场景 + 卖点 → 第一屏就筛掉泛流量**

## 五点描述布局（7条结构）

| 位置 | 内容 | 作用 |
|------|------|------|
| **五点1** | 用户定位 + 主场景 + 核心整体卖点 | 总概括，第一句让精准用户"对号入座" |
| **五点2** | 小场景1 + 卖点1 | 代入感，用户脑补"我也遇到过" |
| **五点3** | 小场景2 + 卖点2 | 继续代入 |
| **五点4** | 小场景3 + 卖点3 | 继续代入 |
| **五点5** | 小场景4 + 卖点4 | 继续代入 |
| **五点6** | 竞品对比 + 选择理由 | 解决最后犹豫 |
| **五点7** | 综合补充信息 | 兜底 |

**核心动作：** 用户的阅读路径 = 跟着文案代入式思考 → "这就是为我设计的"

## 选品岗位读后感（龙哥）

> 选品主要贡献 = 为开发和运营提供精准的"用户×场景×需求"

**问题时刻 = 小场景**：
- 问题越大越无法容忍，越需要解决
- 用户对解决方案的需求程度 = 产品的价值感知程度
- **我们买东西，是为了"雇佣"它完成任务或解决问题**

**提示词（挖掘问题时刻）：**
```
提示词1：挖掘大场景
我要在亚马逊美国站销售一款[产品]，目标用户是[人群]，
请你帮我描绘出这个人群与[产品]有关使用场景，
挖掘出用户最无法容忍的问题有哪些，
按照无法容忍的程度进行排序。

提示词2：挖掘小场景
围绕场景：[具体场景]。帮我清晰简单的描绘出
用户使用产品过程以及价值感知锚点。
```

**⚠️ AI结论必须用真实VOC验证，不能直接用！**

---

## 开发岗位读后感（旭荣）

> "不要为了开发卖点而开发，站在美国消费者的角度，可以解决用户在使用产品的场景下开发的卖点，才是有效卖点"

**两个误区：**
1. 功能堆砌式创新 → 制造差异化而差异化
2. 自我感觉式设计 → 文化投射

**避坑：** 目标市场本土化数据（关键词/评论/竞品）交叉验证，不靠直觉

---

## 价值主张提炼（贯穿主线）

经过用户定位→场景验证→卖点开发，应该能提炼出**一句话价值主张**：

> 案例：针对的士/网约车司机，在夜间场景下，提供一个"无需操作、自动保存、夜间清晰、音频完整、可被平台采信"的行车记录。

**这条价值主张就是选品-开发-运营的主线**，所有动作不偏离这句话。
