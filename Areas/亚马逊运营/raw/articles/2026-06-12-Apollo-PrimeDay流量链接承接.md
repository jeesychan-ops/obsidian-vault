# Prime Day 流量来了，你的链接接得住吗？

> 来源：Apollo_Spark（阿波罗）
> 核心：一套Prime Day旺季5步框架——产品分层→周期预判→结构搭建→投放策略→当天节奏控制

## 产品分层（决定资源投向）

| 类型 | 目标 | 策略 |
|------|------|------|
| 主推爆款 | 争夺市场份额和排名 | 优先获得广告预算、促销资源 |
| 利润款 | 投入产出比 | 精准流量+高转化词，维持盈利 |
| 清仓款 | 库存消化+现金流 | 价格优势获取流量 |
| 测试款 | 验证新品/新场景 | 控制预算，以获取数据为主 |

## 周期预判（3个阶段）

1. **预热阶段**（活动前）— 消费者搜索/收藏/加购，重点不是ROI而是曝光和加购数据积累
2. **爆发阶段**（活动当天）— 购买意图全年高峰，竞品加剧但转化率最高，重点保流量获取和排名稳定
3. **返场阶段**（活动后）— 错过的消费者继续搜索，竞争较低，ROI往往不错

## 广告结构4层

| 层级 | 作用 |
|------|------|
| 核心关键词层 | 大规模曝光和流量入口 |
| 精准长尾关键词层 | 提高转换效率 |
| 竞品流量层 | 获取竞品流量，争夺市场份额 |
| 品牌防守层 | 品牌词+店铺词，防竞品截流 |

## 4种投放策略

| 策略 | 适用 |
|------|------|
| 进攻 | 核心关键词，提高竞价夺优质广告位 → 主推爆款 |
| 防守 | 品牌词+店铺词+核心ASIN流量防截流 |
| 品牌 | SP Brands/Video/Brand Store强化品牌认知 |
| 捡漏 | 长尾词/自动广告/商品投放补遗漏流量 |

## 当天节奏控制

- **监控**：点击率/点击成本/流量增长 → 判断是否符合预期
- **调整**：转化率/广告订单占比/库存消耗速度 → 动态分配预算
- **最后几小时**：最后一波集中爆发，核心活动不能提前耗尽预算
