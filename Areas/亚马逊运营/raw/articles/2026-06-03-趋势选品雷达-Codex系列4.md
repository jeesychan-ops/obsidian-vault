# Codex系列4：每天8点自动选品 Skill

> 核心：Google Trends RSS 抓趋势新闻 → AI提取产品机会 → MCP初筛 → 飞书推送

## 流程

1. 按目标市场读取趋势新闻（Google Trends RSS）
2. 从新闻提取：事件、人群、痛点、场景、关键词、购买需求
3. 转为可验证产品方向（如"高温"→"便携降温风扇"）
4. 调用MCP工具验证：关键词、趋势、竞争、价格带、竞品
5. 每天只推 Top 3

## 卖家精灵 MCP调用顺序（→ 替换为 Sorftime MCP）

原作用卖家精灵，我们用 Sorftime 替代：

| 步骤 | 原工具 | Sorftime替代 |
|------|--------|-------------|
| 趋势验证 | google_trend | Google Trends RSS直接抓 |
| 关键词需求 | keyword_research | keyword_detail + keyword_trend |
| 长尾扩展 | keyword_miner | keyword_extends |
| 市场验证 | market_research | category_report + search_categories_broadly |
| 产品初筛 | product_research | potential_product + product_search |

## 评分规则（100分制）

原作建议将新闻信号和市场数据结合起来评分，防止"看到新闻就兴奋"。

## 飞书推送格式

每条推送至少包含6部分：
1. 新闻信号：为什么今天关注
2. 产品假设：可能带来什么购买需求
3. 关键词：准备验证哪些
4. 数据：搜索量、竞争度、价格带
5. 机会评分：为什么排进Top 3
6. 下一步动作

## 核心原则

- 新闻只当信号，不当上架依据
- 产品必须经过MCP初筛
- 缺数据必须说明，不能编造
- 每天只推Top 3
- 先手动跑3天，再定时每天8点
