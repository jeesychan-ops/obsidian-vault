## 核心内容

### 1. COSMO不是新搜索算法

COSMO（Commerce OS MOdel）本身并不负责商品排序，而是一个**商业常识知识引擎（Commercial Commonsense Engine）**。

它生成的知识被应用于：
- 搜索相关性系统（Search Relevance System）
- 推荐系统（Recommendation System）
- 导航系统（Navigation System）

关键区别：传统搜索解决"用户搜索了什么"，COSMO解决"用户为什么搜索"。

### 2. COSMO的两大数据来源

- **Search-Buy数据**：用户搜索了什么以及最终购买了什么
- **Co-Purchase数据**：哪些商品经常被同时购买

关键细节：**COSMO学习的重点不是商品描述，而是用户行为**。因为卖家可以优化文案，但用户行为很难伪造。

案例：数百万用户搜索"孕妇鞋"后最终购买了防滑鞋 → 系统推断"Pregnant → Require → Slip Resistant"

### 3. 15种关系类型

| 关系类型 | 说明 | 示例 |
|---------|------|------|
| Used For | 用于什么 | Yoga Mat → Stretching |
| Used For Audience | 适合什么人 | Baby Carrier → New Parents |
| Capable Of | 能够实现什么 | Winter Coat → Provide Warmth |
| Used In Location | 在哪儿使用 | — |
| Used On Body | 用于身体哪个部位 | — |
| （以及更多共15种） | | |

### 4. COSMO知识生成流程

用户行为数据 → LLM生成潜在知识 → 规则过滤 → 相似度过滤 → 人工审核 → 知识图谱

注意：Amazon没有完全依赖LLM，论文强调了Rule-based Filtering、Similarity Filtering和Human Feedback三个环节。

### 5. 搜索路径的根本变化

传统：**关键词 → 商品**

COSMO：**目标 → 任务 → 子任务 → 产品类型 → 属性 → 商品**

案例：用户搜索"Camping"
- 系统识别"Camping"是一种活动（Camping Activity）
- 推导出用户可能需要：露营炊具（Camping Cookware）、露营烹饪配件（Camping Cooking Accessories）
- 最终定位到具体商品

### 6. COSMO支撑Rufus/Alexa

AI购物助手的挑战从来不是认识商品，而是理解用户。COSMO承担了Rufus/Alexa背后的推理层角色——将用户身份、目标、场景和担忧转化为具体商品方案。

案例："Baby Safety" → 最终导航到"Breathing Detection Monitor"（因为用户真正关心的是婴儿呼吸安全问题，不是监视器本身）

### 7. 对卖家的意义

过去：**关键词覆盖**
未来：**用户意图表达**

优秀的Listing需要表达：
- 产品为什么存在
- 适合谁使用
- 在什么场景下使用
- 能够解决什么问题

优化的重点从**关键词覆盖率**转向**产品与用户需求之间的关系强度**。
