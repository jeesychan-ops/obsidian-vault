# 把买家评价变成 Listing 弹药: 我用 7 层 VOC 框架

## 核心观点

"看评价"和"反向工程评价"是两件完全不同的事。市面上Helium10/卖家精灵的review insights只做了1层（词频统计），而真正能从评价里挖出卖点需要7层框架。

## 7层VOC框架

### Layer 1: 用户语言 → 标题/5点用什么词
- 卖家和买家说的是两套语言（卖家写"Premium Quality"，买家写"feels sturdy"）
- 把买家的词放进listing，转化率比自创术语高30-50%
- 提取4类表达后打分排序（频次/情感强度/是否新发现/具象度）
- 得分高→标题，中→5点，低→后台search terms
- **精髓：不要"创作"listing，要"搬运"买家的话**

### Layer 2: 购买动机 → 主打礼物还是自用
- 3种心智：礼物心智、自用心智、信仰/身份心智
- 从评价找信号：触发场景、痛点暴露、紧迫性
- 如果60%买家是"买来送人的"，你listing通篇讲自用就错配了
- **精髓：先搞清楚"用户买来干嘛"，再决定listing主打什么心智**

### Layer 3: 使用场景 → 主图拍什么场景
- 物理场景/使用频率/携带方式/配合物品
- 用户会具体列出他们装了什么，你一字不改搬进listing
- 从"空话"到"具象"——直接列出用户列的具体物品清单
- **精髓：用户已经告诉你他们怎么用了，照着写就行**

### Layer 4: 隐藏卖点（ROI最高） → 核心差异化
- 对比listing现有内容：用户提了、listing没写的 → 候选隐藏卖点
- 跨条评价聚类：同一被≥2人独立提到 → 高可信度
- "surprise / wish / didn't expect" → 隐藏卖点信号
- 实例：一个物理结构特征被用户提了2次用了不同说法，listing一个字没写，得分95/100
- **精髓：最大的差异化，藏在你"觉得没必要说"的地方**

### Layer 5: 决策对比 → 定价+对标
- 明确品牌提及、类目对比、价格锚定、升级动机
- 不能提竞品品牌名（红线），但可以用"类目特征"表达差异
- 用户说"价格很值"说明有提价空间
- **精髓：搞清楚用户在和谁比，才知道强调什么、定价多少**

### Layer 6: 微吐槽 → Q&A预设
- "Wish it was..." / "Slightly..." / 5星评价里的"但是"
- 微吐槽=已经发生的退货成本，转成Q&A=预期管理
- 5星里的"但是"比1星更有价值
- **精髓：好评里的"但是"，是免费的退货预警系统**

### Layer 7: 情感金句 → A+情感锚点
- 感叹词/终极判断/后悔欣慰/推荐意愿
- 不能直接引用买家评价（政策红线）
- 改写法：把主语从用户换成产品
- **精髓：把买家的情感，翻译成品牌的声音**

## 洞察合成

7层挖完后，先做"洞察合成"——把7层发现收敛成4个高层结论，再进入listing改写。
